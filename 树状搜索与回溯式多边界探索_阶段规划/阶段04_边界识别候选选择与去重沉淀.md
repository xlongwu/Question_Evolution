# 阶段04：边界识别、候选选择与去重沉淀

阶段编号：04

阶段名称：边界识别、候选选择与去重沉淀

阶段目标：让系统能够判断一个候选是否形成新的能力边界，是否与已有边界重复，以及是否应作为边界叶子保存而不是继续深入。

阶段范围：覆盖评分效果分析、边界轴检测、候选去重和保留策略，不负责维护下一轮 frontier，也不直接扣减搜索预算。

主要任务：

1. 在 `analyze_evolution_effect.py` 中扩展输出 `boundary_axis_detected` 和 `is_new_boundary_for_sample`。
2. 结合 `effect_label`、分数变化、目标 `boundary_axis`、检测到的 `boundary_axis_detected` 和已有 `discovered_boundaries` 判断候选是否形成新边界。
3. 设计 `dedup_signature`，用于识别同一样本内高度重复的边界候选。
4. 在 `candidate_selection.py` 中扩展输出 `selected_into_mainline`、`selected_as_boundary_leaf`、`discard_as_duplicate`。
5. 支持“保留为边界叶子但不继续深入”的候选状态，避免命中边界后只能进入主链或被丢弃。
6. 对目标能力轴和实际检测能力轴不一致的候选保留审计信息，供人工复盘。
7. 明确低质量候选、重复候选、无效降分候选和高价值边界候选的处理规则。

涉及模块/文件范围：`analyze_evolution_effect.py`、`candidate_selection.py`、评分输出文件、`search_graph.jsonl`、`evolution_state.discovered_boundaries`。

前置依赖：依赖阶段01数据契约、阶段03候选 metadata，以及现有 scoring 输出字段。需要确认当前评分结果中是否稳定包含 `score_rate_before`、`score_rate_after`、模型答案质量标签或等价字段。

完成标准：

1. 同一样本可以沉淀多个不同能力轴边界。
2. 重复候选能被标记为 `discard_as_duplicate`，不会反复进入下一轮。
3. 有效边界叶子可以被保存，即使它不继续作为主线扩展。
4. 候选选择输出足以供阶段05更新 `discovered_boundaries`、branch 状态和 search graph。
5. 对边界轴不一致、证据不足和解析失败的候选有明确兜底状态。

测试与验证方式：

1. 构造同轴重复、跨轴新边界、无效降分、无降分但结构性错误等样例。
2. 验证 `dedup_signature` 对重复候选稳定，对不同能力轴候选不过度合并。
3. 检查 `selected_into_mainline` 与 `selected_as_boundary_leaf` 可以并存或互斥的规则是否清晰。
4. 验证 `discovered_boundaries` 增量不会误覆盖历史边界。

风险与注意事项：

1. 边界去重过严会误杀相邻层级细分，过松会让样本池重复膨胀。
2. 不能只用分数变化判断边界，否则会漏掉结构性错误。
3. `boundary_axis_detected` 与目标 `boundary_axis` 不一致时，不应直接丢弃，应保留审计字段。
4. 候选选择阶段不应直接决定样本停止，停止应由阶段05统一处理。

预计交付物：

1. 边界轴检测输出。
2. 新边界判断规则。
3. 去重签名规则。
4. 候选选择扩展输出。
5. 边界叶子保存策略。

Codex Goal 建议：适合作为一次独立 Codex Goal 执行。该阶段是“边界判断闭环”，不建议拆成分析脚本和选择脚本两个孤立 Goal。
