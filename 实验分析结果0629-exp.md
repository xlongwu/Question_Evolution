# 2026-06-29 `exp` 实验结果分析

这次实验一共跑了 5 轮进化，加上 1 轮 baseline。整体看下来，这不是一次“全面开花”的实验，而是一轮很典型的“部分样本真正压出边界，部分样本没有继续压下去”的实验。

最值得先说的结论有三个：

- 这轮实验确实找到了 Qwen 在某些“结论分层”和“证据边界”题上的明显短板，不是靠压分压出来的假结果，而是真实的判断失误。
- 这轮实验也暴露出当前进化 prompt 的一个问题：它很擅长把原题里“问得比较散、比较宽、比较开放”的部分整理成更清楚、更聚焦的问题。但“更清楚”不一定等于“更难”，有时反而会让模型更容易答对。

这里最典型的是 `475`。

- 原题在问的是：“通过哪些视频细节，判断这个人是不是实际驾驶员，而不是乘客？”
- 这种题表面上很专业，但回答空间很大。模型可以不断罗列“上车、系安全带、扶方向盘、踩踏板、看后视镜”这些驾驶细节，看起来说得很多，却不一定真正回答到题目的核心。
- 这道题真正想考的，其实不是“会不会列驾驶细节”，而是“能不能把‘局部视频里确实在开车’和‘整次跨市外出都由他驾驶’这两件事分开”。
- 后来的进化版把题目改成了更明确的样子：先承认“某人在一段视频里确实坐在驾驶位并实施了方向盘、踏板等车辆操控”，再追问“能不能仅凭这些画面，就认定他是整次跨市外出的实际驾驶员，而不是中途短暂入座或与他人换驾后的临时操控者”，最后要求指出“还缺的最小事实是什么”。
- 这一改把题眼钉得很死。原来那些可以靠铺很多驾驶细节混过去的空间被砍掉了，模型不用再猜题目到底要列哪些点，只要回答“局部驾驶成立，但全程实际驾驶员身份还不能成立，还缺连续性闭环或无断点时间轴记录”就行。结果是，Qwen 反而更容易答了，直接拿了满分。
- round 3 到 round 5 基本没有新的有效变化，流程却还在继续跑，说明系统已经具备“能发现边界”的能力，但还不够会“见好就收”。

下面按整体、按轮次、按样本来展开说。

## 一、整体情况

- baseline 轮的平均得分率是 `0.8117`。说明起始的 4 道题里，Qwen 整体答得不差，甚至偏强。
- round 1 后平均得分率掉到 `0.6917`，说明第一次进化是有效的，至少有一部分题确实压出了新的区分度。
- round 2 后平均得分率进一步掉到 `0.5458`，这是这次实验真正的成果轮。因为这一下不是小幅波动，而是有两道题直接把模型从高分拉到了低分。
- round 3、round 4、round 5 的平均得分率完全不再变化，始终停在 `0.5458`。这不是“模型稳定收敛”，更像是“有效工作已经做完了，后面只是把结果重复搬运了三遍”。

从最后结果看，4 个样本分成了三类：

- 一类是真正压出边界的样本：`18318` 和 `859`
- 一类是重写后反而被模型轻松吃透的样本：`475`
- 一类是原本担心它虚高，但一旦改写，反而说明它答得确实不差的样本：`6406`

所以，这轮实验最大的价值不是“平均分下降了多少”，而是把样本分层分得很清楚：

- 哪些题是 Qwen 真会
- 哪些题是 Qwen 其实不会，只是原题写法给了它发挥空间
- 哪些题改写方向选对了
- 哪些题改写后反而把原来的模糊空间抹平了

## 二、每一轮在做什么，结果说明了什么

### Round 0：先看原题，Qwen 到底是“真会”还是“写得像会”

- 这轮的作用就是做 baseline，不做任何进化。
- 四道题里，`859` 直接拿了满分，`6406` 拿到高分，`18318` 刚好在高分阈值附近，`475` 是明显偏低的一题。

这一轮最关键的信息不是分数本身，而是四道题的“高分原因”并不一样：

- `859` 和 `6406` 的高分，表面上都像“答得很完整”，但完整不等于真的打中了题眼。
- `18318` 看起来也不低，但它其实已经在暴露一个危险信号：模型会用很多专业说法把自己包装得像是看懂了，但真正卡人的地方它未必站稳。
- `475` 虽然不是高分，但它也不是完全不会，它更像是“会说一堆驾驶细节，却没有把最关键的排他性闭环讲透”。

也就是说，round 0 不是简单告诉我们“Qwen 强不强”，而是在告诉我们：有些题它是“真会”，有些题它是“会写”，这两者后面会被进化过程一点点分开。

### Round 1：第一轮进化，核心动作是把题目从“开放罗列”压缩成“分层判断”

- 这一轮是整次实验最有代表性的一轮，因为 3 道题都被统一改成了 `O2_subclaim_localization` 风格。
- 这个风格的核心不是让模型多写，而是逼模型先回答一个更难的问题：你到底知道“哪一层已经成立，哪一层还没成立”吗？

这一轮的进化方向很明确：

- `6406` 从“怎么通过时间差判断有无涉毒嫌疑”改成“先拆成两层：`18 分钟明显超出 4 到 6 分钟正常往返时间，所以异常停留已经成立`，但 `仅凭一次超时还不能直接成立毒品交易或藏匿嫌疑`；然后再从三个补充事实里选最关键的那一个。三个补充事实分别是：`进入盲区前曾短暂减速、返回主路后迅速驶离`，`本月内曾 3 次在深夜进入同一盲区且每次停留都明显超时`，以及 `办案人员已依法调取该车全部历史违章记录`。”
- `859` 从“比较正常穿过和盲区内意外的时间规律”改成“先回答：`超过个人历史正常穿过时长上界而仍未出现`，到底已经能支持什么；然后再回答：`这是不是已经足以支持盲区内发生意外`，如果还不够，到底还差什么最小关键事实”。
- `18318` 从“怎么区分意外脱缰和故意放任”改成“视频到底已经支持到了哪一层：是只支持 `管理人客观上控制不力、存在放任表现`，还是已经支持 `管理人主观上故意放任攻击发生`”。

这一轮结果很有意思，三道题走向完全不同：

- `18318` 从 `0.8` 直接掉到 `0.25`。这道题的进化非常成功。原题里模型还能靠“专业、全面、像办案”混过去；一旦逼它分层，它就露出关键错误：它把“视频里能看见管理人在场、犬只持续兴奋前冲、脱离前未明显收紧牵引、脱离后约 5 秒内没有追赶呼喊或隔离”这些只能支持 `客观控制不足或放任表现` 的事实，直接上推成了 `主观故意放任已经成立`。
- 这道题真正难的，不是把分析点讲全，而是守住“现有视频最多能证明到哪一层”这条边界。题目区分的其实只有两步：第一步，`现有视频能不能说明管理人客观上控制不力、存在放任表现`；第二步，`现有视频能不能进一步说明管理人主观上就是故意放任攻击发生`。通俗说，前一步更像“看得出来他没管好、也没及时制止”，后一步才是“能认定他心里就是放着不管，任由攻击结果发生”。原题允许模型把犬只状态、管理人反应、高风险环境、完整时间线都讲一遍，所以即使没守住这条边界，也还能拿到 `0.8`；进化版则直接把这个错误暴露出来了。
- 更稳妥的答法应该是：现有视频更能支持 `客观控制不足、放任嫌疑已经出现`，但还不能直接坐实 `主观故意放任`，因为关键画面被遮挡了，还看不清脱缰瞬间到底是自然脱落、设备问题，还是主动松手或解扣。

- `859` 还是满分。说明第一版重写虽然把题收窄了，但对 Qwen 来说还不够狠。它依然能准确区分“异常滞留已成立”和“意外归因还不能成立”。
- `6406` 反而从 `0.8636` 升到 `0.9333`。说明这道题第一次改写把原题的模糊空间清掉了，模型反而更容易知道该怎么答。
- `475` 这一轮没有动，直接透传。系统当时判断它暂时不像“高分虚高”，所以没急着改。

这一轮说明了一件很重要的事：

- 当前 prompt 最擅长的，不是把所有题都变难，而是把“原题真正要考哪一步判断”变得更明确。
- 一旦原题真正的难点就是“分清 `客观表现已经成立` 和 `更高一层结论还不能成立`”，这个改法很有效，比如 `18318`。
- 但如果原题本来就缺少一点“让模型在几个相近判断里明确失手”的尖锐点，只是改得更清楚，模型反而会更舒服，比如 `6406`。

### Round 2：第二轮进化，开始出现真正的分化

- 这一轮最关键，因为实验真正的成果主要集中在这里。
- 这轮只重点动了两道题：`475` 和 `859`。
- 同时，它也开始出现“不同题应该走不同算子”的迹象，而不是所有题都统一用 O2。

先看 `859`。

- round 1 后，它还是满分，所以系统判断不能继续原样问了。
- 路由上给了 `O1_gap_choice` 作为主算子，`O2_subclaim_localization` 作为备选。
- 最后选中的题，不再只是问“哪一层成立”，而是进一步问：“两个看起来都合理的补充事实里，哪个才是真正把结论推过去的最小关键缺口？”

这一改非常有效。

- Qwen 选了“`盲区从 A 到另一侧是唯一可行通路，中间无岔路、无房门、无电梯口、无可停留点，也不存在被他人接走或绕开的合理可能`”，没有选“`获得盲区内的直接状态线索，例如异常声响、呼救、跌倒画面，或老人本人后来说明自己当时为什么停留`”。
- 它的思路是：先把所有替代路径排干净，才能说更像意外。
- 这听上去很像在讲道理，但恰恰暴露了一个很典型的问题：它把“先排掉岔路、房门、电梯口、可停留点、被人接走或绕开的可能性”当成了最小关键动作，却没有意识到，这道题真正缺的是“拿到盲区内的直接状态线索，把老人当时到底是跌倒、呼救、异常停留还是正常停留直接钉死”这一步。
- 结果这题从 `1.0` 直接掉到 `0.0`。

- 这说明，Qwen 不是完全不懂这题。它知道现在只能先说“异常滞留”，不能直接说“意外已成立”；但一旦题目继续追问“哪个证据才是真正最关键的那一个”，它就会把“排除式补强”误当成“最小关键缺口”。也就是说，它能守住大方向，但抓不住真正决定胜负的关键一步。

再看 `475`。

- 这题从 baseline 就偏低，所以第二轮走的是 `reconstruct_low_score_boundary`，不是高分虚高进化。
- 系统给了两个候选方向：`O8_double_threshold_claim` 和 `O2_subclaim_localization`。
- 最后选中的是 `O2` 版本，也就是把题改成“先承认某人在一段视频里确实坐在驾驶位并操控车辆，再问：这能不能直接认定他就是整次跨市行程的实际驾驶员？”

这题改写后的逻辑其实很干净：

- 第一层：`现有视频已经能证明某人在特定时段里坐在驾驶位并实施了方向盘、踏板等车辆操控`，这一步已经成立。
- 第二层：`能不能仅凭这些画面就认定他是整次跨市外出的实际驾驶员，而不是中途短暂入座或与他人换驾后的临时操控者`，这一步还不能成立。
- 现在真正缺的最小事实是：`未覆盖区间内是否一直由同一人持续坐在驾驶位、且没有发生控制权交接的连续性画面或可无缝衔接的时间轴记录`。

这个改法从“训练题设计”角度看是成功的，因为题眼更准了。

但从“压模型”角度看，它失败了。

- Qwen 这次直接拿满分。
- 原因不是模型突然变强了，而是改写后把原题里的噪声清掉了：原来它可能会沉迷于罗列“左前门上车、系安全带、扶方向盘、踩踏板、看后视镜”这些驾驶细节；现在题目直接先承认“这些局部操控事实已经成立”，模型就很容易顺着答出“还缺的是整段跨市行程里的连续性闭环和无换驾断点”。

所以 `475` 的第二轮结果很值得警惕。这不是一道坏题，也不是一次坏进化；它只是告诉我们：有些样本适合用来“澄清能力点”，不适合用来“继续压分”。

再看另外两题：

- `18318` 已经在 round 1 明确压出边界，所以这一轮直接停了，这是合理的。
- `6406` round 1 进化后分数反而更高，所以这一轮被判成“高分整体可信，不属于明显虚高”，也停了。

这一轮说明：

- 系统开始具备“别把所有题都硬往下压”的意识。
- 真正有效的新边界，主要是在 `859` 这类“还差一刀”的题上挖出来的。
- 但 `475` 也提醒我们，当前 prompt 很容易把题改成“更标准、更干净”，而不是“更难、更能让模型失误”。

### Round 3 到 Round 5：实验还在跑，但新信息已经很少了

- 从 round 3 开始，四道题基本都变成透传。
- `18318` 和 `859` 被标成 `effective_boundary_sample`，不再继续进化。
- `6406` 和 `475` 虽然状态里还写着 `continue_with_new_operator`，但实际 action 已经不再真的启动新的算子探索。
- 分数也完全不再变化。

这几轮说明了两个问题。

第一个问题是好的：

- 系统至少没有在已经有效的样本上乱改。
- `18318` 和 `859` 被停住，说明系统知道“这题已经找到 Qwen 的弱点了，继续改意义不大”。

第二个问题是不够好的：

- 对 `6406` 和 `475` 这类“没压下来，但也没再产生新信息”的题，系统没有及时结束实验。
- 它在状态里已经知道“建议换算子”，还给出了 `O4_near_level_ranking`、`O8_double_threshold_claim` 这些下一步方向，但实际没有落成新的有效尝试。
- 结果就是 round 3 到 round 5 主要是在重复跑透传流程，实验成本继续花，信息增量却接近于零。

所以这几轮本身不是没有意义，它们给了我们一个很明确的优化信号：

- 当前系统会“记住结果”，但还不够会“把记住的结果转成真正的新一轮动作”。
- 也就是说，状态更新已经有了，策略调度还没完全跟上。

## 三、按样本逐个分析

### 样本 `18318`：这次实验里最成功的一题

- 原题是“犬只伤人致死时，怎么从视频回溯判断这是意外脱缰还是人为故意放任”。
- baseline 时 Qwen 拿了 `0.8`，看起来不低。
- 但它的问题在于：表面上它会拆层，也会提缺失事实，可是真正最要命的地方答错了。

它错在哪里？

- 它把“管理人在场、犬只兴奋、没有及时制止”这些视频里可见的行为，直接上推成“主观放任层已经成立”。
- 其实这道题真正考的是：视频最多先支持“控制不足”或者“放任嫌疑”，但不能直接坐实“主观故意放任”。
- Qwen 恰恰在这一步跨过去了。

所以 round 1 把题改成“请只拆成两层，回答哪一层已被支持，哪一层还不能成立”之后，它的错误一下就被放大了。

- 它仍然坚持“主观放任层已被支持”
- 于是得分直接掉到 `0.25`

这道题说明的能力边界非常清楚：Qwen 会讲很多专业话，也会补“看起来很关键”的缺失事实，但它不一定能守住“视频能证明到哪一层”这个边界。这个问题不是知识不够，而是结论往上推得太快。

为什么后面没再继续进化？

- 因为第一轮已经打中了核心弱点
- 再继续改，未必能得到更清楚的新信息
- 所以它从 round 2 开始就一直被当作有效边界样本保留

这题是本次实验最成功的成果之一。

### 样本 `859`：第一刀没压下来，第二刀压得很准

- 原题是“老人进入监控盲区后，怎么从时间规律判断是正常穿过还是盲区内意外”。
- baseline 时 Qwen 拿满分。
- 它的问题不是方向错，而是写得太满，掺进了很多题外工程化扩展，容易显得特别强。

round 1 的进化思路是：

- 把原来的开放比较题，收束成“哪一层已经成立，哪一层还不能成立”。

这个改法为什么没压下来？

- 因为 Qwen 对这个层级区分其实是会的。
- 它能正确说出：现在只支持“异常滞留”，还不能直接说“已经发生意外”。
- 说明它在“第一层边界”上很稳。

round 2 为什么有效？

- 因为这次题目不再只问“你知道边界在哪”，而是进一步问“如果要往意外方向推进，哪个证据才是真正最关键的那一个？”
- 这一下，题目不再只是考“会不会分层”，而是在考“能不能分清关键缺口和一般补强”。

Qwen 在这里犯的错很有代表性：

- 它选了“`盲区是唯一通路、没有岔路、没有房门电梯口、没有可停留点、也不存在被他人接走或绕开的合理可能`”这一项，也就是先把其他可能路径都排掉。
- 但题目真正想要的是“`盲区内的直接状态线索，例如异常声响、呼救、跌倒画面，或老人本人后来说明自己当时为什么停留`”这一项，也就是直接去碰“盲区里到底发生了什么”。

这说明它的思路很像人工推理里常见的一个偏差：

- 它知道“把其他去向和绕开可能性排干净”很重要
- 也知道“拿到盲区内直接状态线索”很强
- 但它没分清“重要”和“最关键”不是一回事

最后这题从 `1.0` 掉到 `0.0`，而且不是乱扣分，是因为核心判断整个反了。

这题说明的边界也很清楚：Qwen 能做“先区分成立层和未成立层”，但在“从两个看似都合理的补强事实里挑那个真正决定性的”这一步，会有明显失手。这也是本次实验最有价值的第二个成果。

### 样本 `475`：题目变准了，但模型也更容易答了

- 原题是“从视频里看哪些细节，才能判断这个人是实际驾驶员而不是乘客”。
- baseline 分数不高，只有 `0.5833`。
- 乍看会觉得这题已经不强了，好像不用再动。

但系统在 round 2 还是对它做了“低分边界重构”，这一步本身是有道理的：

- 原题偏开放，容易被答成“驾驶细节大清单”
- 真正的难点其实不是“看没看到人在开车”，而是“能不能排除只是临时开了一段，或者中途换驾”

重写后的题把这一点说得非常直白：

- `某人在一段视频里确实坐在驾驶位并操控车辆` 这一步已经成立
- 但 `能不能直接认定他是整次跨市出行的实际驾驶员` 这一步还不能成立
- 还缺的是连续性和排他性

从题目设计角度说，这次改写是很好的。

- 它比原题更聚焦
- 更少噪音
- 更能让人看见真正要考什么

但结果却是 Qwen 直接拿了满分。

这说明什么？

- 说明 `475` 原来并不是“Qwen 完全不会”
- 它更像是“原题容易让它往驾驶细节堆砌方向跑偏”
- 一旦题目把“某人在一段视频里确实坐在驾驶位并操控车辆”先垫好，模型很容易顺着答出“还缺的是整段跨市外出里有没有换驾断点、有没有连续时间轴闭环”

所以这题的结果不是失败，而是在提醒我们：当前进化 prompt 很擅长把题目“讲清楚”，但“讲清楚”不总等于“更能压模型”。

对这类题，下一步如果还想继续压，不能再只做“层级拆分”，而要逼模型在几个接近但不等价的闭环证据里做更细的比较，比如：

- 哪种连续性才是真正排除换驾的最小闭环
- 哪些片段即使连续，也仍然不足以排除中途交接
- 哪种“看起来像闭环”的证据，其实还是留了漏洞

### 样本 `6406`：系统怀疑它虚高，但改写后反而证明它答得真不差

- 原题是“涉毒车辆进入监控盲区，怎么通过时间差判断是否存在交易或藏毒嫌疑”。
- baseline 分数 `0.8636`，当时系统怀疑它是“写得太满超题”。

这个判断不是没道理。

- baseline 里的 Qwen 答案确实很像一种常见情况：写得又全又稳，法律、程序、协同、补强都提到了，但真正题眼不一定压实。

所以 round 1 把它改成了一个更尖锐的题：

- 先拆成两层：`18 分钟明显超出正常 4 到 6 分钟往返时间，异常停留已经成立`，但 `仅凭这一回合的超时和盲区内没有正常目的地，还不能直接把结论上推成毒品交易或藏匿嫌疑已经站稳`。
- 再从三个候选补充事实中选最小关键补强项：是“进入前短暂减速、返回后迅速驶离”，还是“本月内 3 次深夜进入同一盲区且每次都明显超时”，还是“已依法调取全部历史违章记录”。

按理说，这样应该更容易让“泛泛而谈”的答案露馅。

但结果没有。

- Qwen 不但没掉分，反而分更高了
- 它很准确地说出了：`18 分钟超时和盲区无正常目的地` 这一组事实已经足以支持“行为异常”，但还不足以支持“特定涉毒指向”；如果要补，最关键的是“本月内 3 次深夜进入同一盲区且每次都明显超时”这种重复模式化异常，而不是“短暂减速后迅速驶离”或“已经调取历史违章记录”。

它只丢了一点细分项分，主要是没有把“18 分钟明显超过 4 到 6 分钟”和“盲区无正常目的地”这两个题干事实咬得更死。

这题的结果其实很有价值，因为它帮我们排除了一个常见误判：不是所有“写得很满”的高分答案都是虚高，有些题 Qwen 确实已经理解得比较到位。所以 `6406` 不是一次失败进化，它更像一次“验证性进化”：改写之后没有压下来，反而说明 baseline 的高分大体可信。

这题也说明当前 prompt 的一个局限：

- 如果模型本来就会这种“先承认 `异常停留已经成立`，再否认 `特定涉毒指向已经成立`”的分层判断，那把题收束成两层后，反而会帮它答得更稳

下一步如果还想继续动这题，应该换方向，不要再继续只问“哪一层成立”，而应该去逼它比较：

- 是不是所有超时都能上推到“特定涉毒模式”
- 哪种补强事实只是让“异常”更异常，哪种才真的让“涉毒指向”变稳
- 多个候选补强事实之间，哪一个才是决定性而不是陪衬

## 四、没有发生进化的样本，也要怎么看

这次实验里，“没有发生进化”不能简单理解成“没有价值”。

- `475` 在 round 1 没进化，是因为系统当时判断它不是典型高分虚高。但它后来在 round 2 还是被当作低分边界样本重构了，而且结果很有启发。
- `18318` 和 `859` 在后续多轮里都没有再进化，不是因为它们不重要，反而是因为它们已经把 Qwen 的边界暴露得很清楚，再改价值不大。
- `6406` 后面没有继续进化，说明系统认为它已经不属于“值得继续压”的对象。这个判断基本是对的。

真正的问题不在于“有没有继续进化”，而在于：

- 对于已经找到清晰边界的题，系统能停，这是好事。
- 对于没有找到新边界、但状态里还写着建议换算子的题，系统没有真正完成“换算子再试”，这才是后半段的主要问题。

## 五、这次实验到底说明了 Qwen 的哪些能力边界

如果把这次实验里最有价值的信号提炼出来，我觉得至少能看到下面几条边界。

### 1. 它容易把“已经可疑”过早上推成“已经成立”

- `18318` 最明显。
- 视频里已经出现了高风险征兆和不作为，但 Qwen 直接把这当成“主观放任层已成立”。
- 这说明它在从客观表现推主观结论时，容易上推过快。

### 2. 它能守住大方向，但不一定能抓住最关键的补强事实

- `859` 最明显。
- 它知道“现在不能直接认定意外，只能先说异常滞留”，这个方向没错。
- 但一旦问它“最关键的补强是什么”，它会把“盲区是唯一通路、没有岔路房门电梯口、没有可停留点、也不存在被接走或绕开的合理可能”这种很重要但不最关键的信息，当成真正决定性的那一个；反而没有抓住“异常声响、呼救、跌倒画面、老人本人后来说明停留原因”这种能直接碰到盲区内状态的事实。

### 3. 它对“层级拆分题”并不天然弱，甚至有时适应得很快

- `6406` 和 `475` 都说明了这一点。
- 只要题目把层级关系写清楚，它常常能顺着走对。
- 所以单纯把开放题改成“分层题”，并不总能继续压出新边界。

### 4. 它很容易在原题开放时靠“说得全面”保分

- `859` baseline 是满分但有虚高嫌疑。
- `6406` baseline 也是高分，虽然最后证明大体可信，但它同样带有“写得很足”的特征。
- 这说明如果题目本身允许大段发挥，Qwen 经常能靠结构、术语和完整感把分数抬起来。

### 5. 它并不是知识不够，而是“证据到结论”的最后两步更容易出错

- 这次掉分最明显的两道题，都不是冷知识题。
- 它们考的不是“知道不知道这个领域”，而是“知道现有事实最多能推出到哪”。
- 所以这次实验更像是在测 Qwen 的“边界控制能力”，而不是它的知识面。

## 六、从这轮实验能得出的总判断

这轮实验总体是成功的，但成功点比较集中，不是全面成功。

- 成功的地方在于，系统确实找到了两条很清楚的 Qwen 能力边界，分别是 `18318` 和 `859`。
- 不够成功的地方在于，另外两道题没有进一步形成有效压测，而且 round 3 到 round 5 没再产出新信息。

如果用更直白的话说，这次实验证明了 question evolution 这条路是能走通的。它不是只会把题改得更花哨，而是真的能把模型的错误逼出来；但现在还不够稳定，更像“有时能精准命中，有时只是把题改得更规整”。

所以，本轮实验的最大价值不是“平均分从 0.81 降到 0.54”这件事本身，而是我们已经知道哪类改写会真的触发 Qwen 失误，哪类改写只会帮它答得更整齐。这比单看分数下降更重要。

## 七、对当前实验结果和当前 question evolution prompt 的下一步优化建议

下面这些建议分成“流程层面”和“prompt 层面”两类。

### 先说流程层面

- 要加一个“无新信息提前停止”机制。现在 early stop 只看平均分有没有低于 `0.5`，这太粗了。像这次 round 3 到 round 5 分数完全不变，其实早就该停了。
- 要真正把 `recommended_next_methods` 用起来。状态里已经连续给出 `O4_near_level_ranking`、`O8_double_threshold_claim`，但后面没有真正切过去，这让 memory 和 state 只起到了记录作用，没有真正驱动下一轮。
- 对“有效边界样本”可以单独沉淀，不必继续跟全队列一起跑。像 `18318` 和 `859`，从打中之后就应该进入“冻结观察”或“导出样本池”，而不是继续每轮透传。
- 对“改写后得分更高”的题，要单独分类。像 `6406`，它不是失败题，而是“验证高分可信”的题。这类题后面应该换目标，不是继续压分，而是考虑是否作为稳健样本保留。

### 再说 prompt 层面

- 当前最常用的 `O2_subclaim_localization` 已经证明有用，但也已经看到上限。它适合把开放题收束，不一定适合继续往下压，后面不能总是 O2 起手。
- 对像 `859` 这类题，说明 `O1_gap_choice` 很有潜力。因为它不是再问“边界在哪”，而是逼模型在多个候选补强事实里挑那个真正最关键的，比如这次就是逼它在“唯一通路、无岔路、无房门电梯口、无可停留点、无被接走绕开可能”和“盲区内直接状态线索，如异常声响、呼救、跌倒画面、老人本人说明停留原因”之间做选择。这类 prompt 值得继续扩。
- 对像 `475` 这类题，下一步不要再问“哪一层成立”，而应该问“哪些连续性证据看起来像闭环，但其实还不足以排除中途换驾”。也就是把比较对象做得更接近，逼模型分细差别。
- 对像 `6406` 这类题，下一步最好不用“分层定位”了，而改成“近似补强项排序”或者“多个补强项里谁最能让结论上升一层”。比如逼它比较：“进入前短暂减速、返回后迅速驶离”“本月内 3 次深夜进入同一盲区且每次都明显超时”“已依法调取全部历史违章记录”这三种补充事实里，到底哪一个只是让‘异常’更异常，哪一个才真正让‘涉毒指向’更稳。因为它现在已经会基本分层了。
- 对像 `18318` 这类题，后续可以尝试把“客观控制不足”“主观放任嫌疑”“主观放任成立”这几个非常接近的层次再细分，看看模型是不是在所有相邻层级间都会上推过快。

### 最后说一句更核心的 prompt 优化方向

这轮实验说明，当前 prompt 最大的优点是“把题讲清楚”，但下一步更需要的是“把相邻层级之间的差别卡得更紧”。也就是说，后面不要只做“把开放题改成边界题”，而要更多做“把两个都看起来有道理的答案摆在一起，逼模型分出那个更关键、更小、更决定性的差别”。只有这样，才能减少“题目变清楚了，所以模型更会了”的情况，更多得到“题目虽然也清楚，但仍会在关键一刀上犯错”的结果。

## 八、一个简短的收尾判断

如果只问这轮实验值不值得，答案是值得。它已经证明系统能找出 Qwen 的真实边界，也已经说明当前最有效的方向，不是继续扩写，不是继续堆专业术语，而是更狠地逼它回答：像 `18318` 里“客观控制不足或放任表现”与“主观故意放任成立”到底哪一步成立，像 `859` 里“唯一通路、无岔路房门电梯口、无可停留点”和“盲区内直接状态线索”之间，到底哪一个补强事实才是真正关键的那一个。

如果只问这轮实验够不够成熟，答案是不够。真正有效的信息主要出在 round 1 和 round 2，后面几轮更多是在重复已有结果。下一步最该优化的，不是再多跑几轮，而是让“什么时候换算子、什么时候停、什么时候转成更细的比较题”这三件事变得更聪明。


## 优化的prompt

下面这个版本，不是继续把题目统一往“边界判断题”上推，而是把“先判断这道题下一步该往哪种失误点上压”写进 prompt 本身。它的目标不是单纯把题变得更清楚，而是尽量把题改成“两个看起来都像对，但只有一个真的关键”的形态，或者改成“几个很接近的层级里，模型最容易往上推错”的形态。

```python
QUESTION_EVOLUTION_PROMPT_TEMPLATE_V4 = """
# 角色
你是一位负责“能力边界定向压测”的评测题目设计专家。你的任务不是把题目机械改难，也不是把题目改成长、多步骤、强格式的结构题，而是要把原题升级成一道：
1. 单主轴；
2. 可回答；
3. 可稳定评分；
4. 能让模型在一个可归因的关键判断点上暴露错误
的新题。

# 核心目标
请分析原题、参考答案、候选答案和现有评分标准，生成一道升级后的新题，使其满足：
1. 仍然考查原题的核心领域、核心事实和核心能力，不改变题目主题；
2. 新题的主要失分点集中在 1 个关键判断上，而不是因为任务太多、格式太复杂、篇幅太长而失分；
3. 新题优先考以下几类真实短板，而不是泛泛增加复杂度：
   - 分不清“已经异常”与“已经能写成结论”；
   - 分不清两个都像有用的补强事实里，哪一个才真正推动结论上升一层；
   - 抓住了显眼事实，但漏掉真正决定定性的那一步；
   - 看起来知道“不够”，但说不准最少到底缺哪一条；
   - 把“看起来像闭环”误当成“已经形成闭环”；
4. 后续 judge 能根据 rubric 稳定判断对错，而不是靠术语密度、结构完整性或格式服从度打分。

# 第零步：先判断这道题下一步该往哪种失误点上压
在内部先做判断，但不要把全部中间推理输出：

1. 先判断原题当前更像哪一类：
   - 开放题太散，模型靠泛泛扩写保分；
   - 边界已经比较清楚，但模型还没被逼到真正关键缺口；
   - 题目如果继续“讲清楚”，反而会更容易答；
   - 题目已经能稳定暴露边界，继续改信息增量很低。

2. 再判断这道题当前最值得压的主错误是哪一种，只能选 1 个：
   - A. 泛泛罗列，缺少题干事实绑定；
   - B. 只会说大方向，不会指出真正关键缺口；
   - C. 混淆层级，把线索/异常/嫌疑直接上推成结论成立；
   - D. 把两个都重要的补强事实，错认成同等关键；
   - E. 把“看起来像闭环”误当成“已经排他成立”；
   - F. 被显眼信息带偏，漏掉真正决定结论的主轴。

3. 如果你发现“把题改得更清楚”会让候选答案更容易顺着答对，那么禁止继续沿着“先承认第一层，再问还缺什么”的方式改写，必须换题型。

# 第一步：先拆结论，再决定是否继续考这一层
1. 如果原题目标结论可以拆成 2-3 个相邻层级，先拆开。
2. 只能选择其中 1 个最值得压测的相邻层级差异作为主轴。
3. 如果候选答案已经能稳定回答“哪一层成立、哪一层不成立”，本轮不得继续复用同类问法，必须改成更细的比较题。
4. 如果新题会把真正难点提前说破，只剩一个显而易见的缺口可答，则这次改写无效，必须重做。

# 第二步：题型选择
从下面题型里只选 1 种主方式；如必须组合，只允许“1 个主方式 + 1 个轻量辅助方式”。

1. 候选缺口二选一
   适用：模型已经知道“大方向还不够”，但分不清哪个补强事实才是真正关键。
   目标：逼它在两个都像有帮助的补强事实里，选出真正推动结论上升一层的那一个。

2. 子判断定位
   适用：模型容易把“已支持层”和“未支持层”混在一起。
   目标：逼它指出到底是哪一个子判断还不能成立，以及最少还缺什么。

3. 近似项分层
   适用：模型会做粗分层，但不会分更细的相邻层级。
   目标：给 2-3 个都“看起来不能直接下结论”的近似项，要求继续分高低，而不是只做可写/不可写二分。

4. 伪闭环识别
   适用：模型容易把“局部成立”误写成“整体连续成立”。
   目标：让模型区分“局部画面已成立”“连续性仍未闭环”“排他性仍未闭环”。

5. 补强项升级判断
   适用：模型知道某事实会让异常更强，但不会分“更异常”和“结论上升一层”。
   目标：让模型判断哪个补充事实只是让原判断更稳，哪个才真正改变结论层级。

6. 单变量反事实
   适用：只差一条条件变化就能看出模型会不会重新排列判断顺序。
   目标：只改 1 个变量，逼模型说明哪一层变化、哪一层不变。

# 第三步：反复题型与“讲清楚反而变简单”检查
生成新题前必须逐项自检：

1. 这次新题是否仍然只是“哪一层成立 + 还缺什么”的换壳版本？
   如果是，且候选答案已经能稳定处理这一类题，则必须换题型。

2. 这次新题是否把真正难点提前说破了？
   如果候选答案只要顺着题面复述“第一层成立、第二层不成立、缺连续性/缺直接状态/缺排他闭环”就容易拿高分，则必须重做。

3. 这次新题是否只是把题面写得更标准、更聚焦，但没有制造两个足够接近的竞争判断？
   如果是，这种改写大概率只会让题更容易答，不应采用。

4. 如果存在两个以上都说得通的最小缺口，不得做开放式“还缺什么”，必须改成候选项区分题。

# 第四步：复杂度预算
1. 新题必须只有 1 个清晰主轴。读者应能用一句话说清“本题到底考什么”。
2. 新增事实或场景条件最多 3 条。
3. 输出任务最多 2 个，优先 1 个。
4. 候选项最多 3 个；如果是缺口比较题，优先 2 个近似候选。
5. 不得要求大表格、多层标签体系、固定句数、复杂编号系统。
6. 不得把难度主要建立在长篇格式、繁琐约束、复杂任务编排上。
7. 如果上一轮已经很长，本轮优先收主轴，不要继续加材料。

# 第五步：可回答性与唯一失分点检查
1. 题干必须提供完成任务所需事实，不得要求题外知识才能回答。
2. 新题应让弱模型主要在 1 个核心错误上失分，而不是在多个任务点上同时失分。
3. 不得设置只能靠猜、靠经验、靠题外法律评价才能作答的陷阱。
4. 如果参考答案需要大量新增知识才能回答，说明改写失败。
5. 如果删掉题目中的一小段要求后，核心考点明显变化，说明本题混入了多个能力点，必须继续简化。

# 禁止的进化方式
1. 禁止把难度主要建立在题目更长、任务更多、格式更复杂上。
2. 禁止连续多轮只复用“最小关键事实/最小前提/最小跳步/哪一层成立”这一类同家族题型。
3. 禁止把开放题统一改写成边界题后就停住，不再继续追问真正决定胜负的差异。
4. 禁止让题目主要考“遵循复杂指令”，而不是考原题对应的专业判断能力。
```

### 为什么这版 prompt 更适合这轮实验暴露出来的问题

- 它补上了“题目变清楚了，但不一定变难”的自检。这一轮里 `475` 最典型，题目越改越清楚，结果模型越容易答。新 prompt 明确要求：如果改写只是把难点提前说破，只剩一个显而易见的缺口，就必须重做。
- 它不再默认“边界拆分题”是通用终点。`18318` 说明 `O2_subclaim_localization` 这类题型在某些样本上非常有效，但 `6406` 和 `475` 也说明，继续沿这一方向改，常常只会让题更标准、更好答。新 prompt 因此先要求判断“这道题下一步该往哪种失误点上压”，而不是先入为主继续拆层。
- 它把“两个都像对，但只有一个真关键”的设计放到了更靠前的位置。`859` 这一轮真正压出边界，不是因为继续追问“哪一层成立”，而是因为逼模型在“唯一通路、无岔路、无房门电梯口、无可停留点、无被接走绕开可能”和“盲区内直接状态线索，如异常声响、呼救、跌倒画面、老人本人说明停留原因”之间做选择。新 prompt 因此把这类题型提升为优先策略，而不是偶尔使用。
- 它专门补了“伪闭环识别”和“补强项升级判断”两类题型。这是因为 `475` 和 `6406` 已经说明，很多题并不是不会分层，而是不会分“局部成立”和“整体排他闭环成立”，也不会分“异常更强”和“结论上升一层”。
- 它把“重复题型检查”写死了。当前实验后半段已经出现：系统知道要换方向，但题面没有真正切换，信息增量变得很低。新 prompt 至少在单题生成层面，明确禁止继续用同家族问法换壳复用。

### `QUESTION_EVOLUTION_PROMPT_TEMPLATE_V1` 的不足之处

这里说的不足，不是指它完全无效。恰恰相反，这版 V1 已经比更早那种“越改越长、越改越多任务”的 prompt 收敛得多了。但结合这轮实验，它仍然有几个很明确的问题。

- 第一，它仍然过于偏向“先拆结论、再定位子判断、再问最小缺口”这一条主路径。这个方向在 `18318` 上很成功，但在 `475` 和 `6406` 上都说明，它不是通用解法。模型一旦已经会回答“哪一层成立、哪一层还不成立”，继续沿这条路改，反而可能帮它答得更稳。
- 第二，它虽然写了“如果上一轮已满分，优先使用候选缺口二选一、子判断定位、近似项分层”，但还没有把“如果这次改写只是把题变清楚，模型就更容易答对”这个风险单独拎出来做强约束。`475` 正是这样：题眼更准了，但压测效果反而更差。
- 第三，它还没有把“伪闭环”和“真假补强升级”当成独立题型处理。`475` 的难点不是简单缺连续性，而是“哪些证据看起来像闭环，但其实还不足以排除中途换驾”；`6406` 的难点也不是“还缺什么”，而是“哪些补强只是让异常更异常，哪些才真能让涉毒指向上升一层”。V1 里这些都还容易被吸收到 O2 或 O1 里，导致题型表达不够稳定。
- 第四，它的“去重复约束”还不够强。V1 已经意识到不能反复问“最小关键事实”，但这轮实验说明，仅仅提醒“不要换壳复用”还不够，因为模型仍可能把题面改成同一家族的近似表达，结果从 round 3 到 round 5 基本没有新信息。
- 第五，它缺少一个更硬的“清晰化陷阱检查”。现在 V1 更强调“单主轴、可回答、可稳定评分”，这很重要，但还不够。因为有些题恰恰是在被收束成单主轴之后，难点被直接暴露出来，候选答案只要顺着题面复述就容易高分。这个问题在 `475` 上最明显。
- 第六，它对“什么时候不该继续沿边界题方向改”说得还不够死。`859` 的结果已经说明：当模型已经会区分“异常滞留已成立”和“意外归因还不能成立”时，下一步真正值得问的，不再是“边界在哪”，而是“哪一个补强事实才是真正决定胜负的那个”。V1 虽然给了 O1/O4 的空间，但这个转向条件还不够强。

### 总结一句

如果把这轮实验的经验压成一句话，那就是：下一版 prompt 不能只擅长“把题讲清楚”，还必须擅长“在两个看起来都对的答案之间，逼模型选错真正关键的那一个”。`18318` 说明“分层拆开”是有效工具，`859` 说明“关键缺口竞争”更能继续往下压，`475` 和 `6406` 则提醒我们，如果只会把题改得更规范、更清楚，模型不一定更容易出错，反而可能答得更稳。
