## 结论摘要

这组真正进化的样本有一个很明显的问题：题目一轮比一轮长，要求回答的事情一轮比一轮多，但 Qwen 大多数时候还是高分。说明原始问题进化 prompt 的主要作用，不是把题目改到真正容易出错的地方，而是把一题变成多题，把一个判断点变成一串判断任务。

少数样本确实出现了压分，最典型的是 `801`。但它最后压分，也主要是因为题目把“先选最关键缺口、再比较其他缺口、再补最少证据”这些任务叠在一起了，而不是只盯住一个清楚的短板来打。

观察得到的现象：

1. 原始问题进化 prompt 会持续拉长题目，且几乎是单调膨胀：

   - `7337`: `59 -> 1708`
   - `8638`: `52 -> 2415`
   - `14865`: `59 -> 1280`

这说明原始问题进化 prompt 更擅长把题面做大，而不太擅长把问题问尖。

## 逐题分析：真正进化的样本

### 1. `801`

- 轨迹：`1.00 -> 1.00 -> 1.00 -> 1.00 -> 0.95 -> 0.88`
- 题长：`50 -> 2194`

这一题原始问法很简单，只是问：如果视频里只看到嘴巴动了，或者只看到有人用手指点了一下试卷，这样的画面在固定证据时有哪些风险。到了 `round_1`，题目开始从“列风险点”变成“顺着结论能不能成立，一层一层往下拆”。它不再只问“有风险吗”，而是开始追问：

- 你到底能不能从这个动作看出说了什么、指了什么；
- 有没有别的正常解释，而不一定是在传递答案；
- 就算真有交流，能不能证明考生听懂了、采纳了，还改了对应答案；
- 如果现在证据还不够，最需要补什么，判断才会明显改变。

这一轮的变化还是比较清楚的：题目从“泛泛谈风险”，变成了“按结论成立的链条逐步检查哪里站不住”。

但从 `round_2` 开始，它继续进化的方向就不只是“把这个关键判断问清楚”，而是不断把题目做成更长、更绕、步骤更多的分析题。更具体地说：

- `round_2` 开始，题目把场景补得更具体了：除了原来那段无声远景视频，还额外给出“动作后 3 分钟内改了两道选择题”和“监考人员解释自己是在提醒不要漏填姓名、检查答题卡位置”这两项材料。然后它要求把整个判断拆成四个层次去说：动作内容到底能不能还原、有没有合理替代解释、能不能证明考生就是据此改了对应答案、如果再多出什么证据会明显改变判断；最后还追问一个变体：如果视频能看清具体点的是哪道题，但依然没有录音，哪些地方会被强化，哪些地方还是不能直接成立。
- `round_3` 又继续加约束。它不再满足于四层展开，而是要求你必须把“看到动作”到“认定作弊成立”之间的链条压成 4 个必要环节，并逐个写清：哪一环具体证明什么、哪一环最容易被误读、如果只缺这一环会导致什么降级。同时，它还要求从这 4 环里挑出最致命的两个缺口，并进一步回答：如果考生改后的答案恰好都改对了，这会补强哪一步、又补不动哪一步。
- `round_4` 再往前走，就不只是问“现有证据哪里不够”，而是要求你在两套完整说法之间做比较：一套说法是“这些动作已经足够支持有人在传答案”，另一套说法是“这些动作还完全可能是正常提醒，证据还差得很远”。题目要求你把两边各自最强的说法都摆出来，再解释为什么哪一边目前更站得住；此外还要把几个关键缺口按轻重排序，并说明如果误把某个次要缺口当成最关键缺口，会怎样把判断带偏。
- `round_5` 最后一轮继续把任务叠高：它要求先从几个看起来都很严重的问题里，硬选一个“最卡住结论”的主问题；再说明另外两个问题为什么虽然也重要，但暂时排在它后面；再往下追问，如果现在只允许你补一类新证据，最应该优先补什么；最后还要求做反事实比较，比如如果能看清题号、如果能看清改题先后、如果能证明改后答案都改对，分别会把结论往前推进到哪一步。

也就是说，这题最后之所以从满分压到 `0.88`，不是因为它终于只问中了一个很尖的点，而是因为它把好几件事绑在一起问了：先问哪里最缺证据，再问其他缺口为什么没那么关键，再问如果只能补一类证据该补什么。

Qwen 在 `round_5` 真正丢分的关键点，不是把整道题的方向答反了，而是第一步没有先抓住“现有画面最不能证明的，其实是考生有没有真的接收到提示、并据此改了对应答案”。它更先去谈“动作本身可能还有别的解释”。这当然也是问题，但不是题目当时要求先选出的那个最关键问题。

因此，这个样本更像是在测：当一道题里同时摆出几个都像问题的缺口时，Qwen 不总能先抓准最卡结论的那个。它不是不会说“证据还不够”，而是容易把“也有问题”错当成“最关键的问题”。

### 2. `7337`

- 轨迹：`0.80 -> 0.96 -> 0.98 -> 1.00 -> 0.87 -> 0.97`
- 题长：`59 -> 1708`

原题本来只是“除了面貌和衣着，还需要结合哪些因素综合分析，以防跟丢目标”。到 `round_1` 时，它被改造成“地铁换乘站短暂脱视后，如何按优先级重新锁定目标，并指出哪些信息看似有用但不应作为主要依据”。这一步已经把一般性知识题变成了现场策略题。

之后的几轮变化继续沿着“场景更具体、策略更复杂”的方向推进：

- `round_2` 把题目改成了一个更具体的地铁换乘站短时脱视场景：目标最后一次出现时的体态和动作都被写得更细，随后给出三个分流方向和三名侦查员的回报。题目要求你不能泛泛说“都看看”，而是必须在三路里排出谁最该先追、谁只配备份、谁应当放后；还要把信息分成“硬证据、软线索、干扰项”，并解释各自权重。
- `round_3` 又加了一层现场指挥限制：未来 15 秒内，你只能下一次主追指令和一次备份指令，不能三路平均铺开。题目还要求给出一条全组统一口令，并说明这条口令为什么能压住“换外套、像身形”这种最容易误导人的线索；同时还得给出一个双条件触发标准，说明什么情况下主追要降级、备份要顶上来。
- `round_4` 继续把干扰信息加进来，例如其中一路人手里拿着热饮、另一路人没有任何回头或找位动作、还有一路人在并入人流前整理了一下肩带。题目要求你重新区分哪些细节只是看起来像，哪些才是真正能延续上一秒钟身份链的东西；并且要指出如果主追最后追错，最可能是被哪一种细节带偏。
- `round_5` 最后又把时间窗口从“脱视 4 秒”改成了更难的“脱视 14 秒”，要求你比较：时间一拉长后，原来不够硬的信息为什么会开始升权；原来很重要的信息为什么会降权；原来的口令要不要改；双条件触发标准里到底哪一条必须升权、哪一条必须降权。

可以看到，这题每轮都在加：

- 新场景细节
- 新干扰信息
- 新调度约束
- 新反事实比较

但得分轨迹却是 `0.80 -> 0.96 -> 0.98 -> 1.00 -> 0.87 -> 0.97`，说明 Qwen 对这类题其实越做越顺。即使 `round_4` 掉到 `0.87`，下一轮也马上回到了 `0.97`。

这说明这题后面的变化，更多是在不断往里塞新情况，比如又多了分流方向、又多了队员反馈、又多了时间限制、又多了干扰信息。题目看起来更忙了，但没有稳定逼出一个清楚的错点。Qwen 如果有失分，也更像是“主追和备选谁放前面”这种局部排序波动，而不是不会判断应该重点追谁。

### 3. `8638`

- 轨迹：`0.80 -> 1.00 -> 1.00 -> 1.00 -> 1.00 -> 1.00`
- 题长：`52 -> 2415`

这一题是“复杂度膨胀但几乎没有区分收益”的最典型样本。原题只是：“为什么建议将图像保存为 PSD 而不是高压缩 JPG”。到 `round_1` 时，它已经变成了“JPG 作为唯一工作底图是否可行、JPG 在什么边界条件下可以使用、误存高压缩 JPG 在彩印和法庭质证中会放大什么风险”。

随后每一轮都继续加场景和约束：

- `round_2` 开始，这题不再只问“为什么不要把高压缩 JPG 当底图”，而是把工作场景补得很实：假设你在图像上反复增删箭头、比例尺、放大框和文字说明，而且同一张图还要同时用于鉴定书正文、庭审展示和内部流转。题目要求你分别解释：JPG 当唯一底图哪里会出问题、哪些用途受影响最大、什么情况下还勉强能用。
- `round_3` 又把焦点推进到“工作母版怎么管”上。它开始要求你不只是谈文件格式，而是要说明：如果一张底图后面要改很多次标注、还要导出多个版本，PSD 这类可分层母版到底帮你避免了什么；如果一开始就只存高压缩 JPG，后面哪些改动会越改越糟。
- `round_4` 再继续把题目推进到“同一工作底图要对应多种输出版本”的工作流里，例如一份给正文、一份给庭审、一份给内部流转，而且要在短时间内来回切换。题目开始追问的是：如果只剩高压缩 JPG，怎样会影响这些版本之间的一致性、怎样会在领导审签或出庭展示时放大争议。
- `round_5` 最后一轮已经明显变成一个大杂烩式长题了：它不只问 PSD 和 JPG 的边界，还开始要求比较几种不同做法谁更稳妥、不同受众该看哪一版、同一底图怎样在多个任务里周转、哪些风险是对外展示更严重、哪些风险是内部流转时就已经埋下来的。

题目从“为什么不要把高压缩 JPG 当工作底图”这个本来很清楚的问题，一路被扩成了“不同用途怎么用图”“多个版本怎么切换”“领导审签和庭审展示怎么处理”这样的混合题。

但 Qwen 全程都是满分。这说明这道题后面的变化，并没有把它推到一个更容易答错的关键判断点上，而只是不断加使用场景。核心问题还是那个核心问题，只是外面包了越来越多工作流和版本管理的内容。

### 4. `14865`

- 轨迹：`1.00 -> 1.00 -> 1.00 -> 1.00 -> 1.00 -> 0.92`
- 题长：`59 -> 1280`

这一题的原始问法是：“如何通过监控判断‘强行脱拽’猥亵，而非朋友间醉酒打闹”。`round_1` 时，它被改造成“先看什么、再排除什么、最后如何形成倾向性判断”，并要求给出两个最关键的支持特征和动摇条件。这一步还是比较合理的，因为它把题目从常识性判断推进到了链式研判。

但后续每轮还是继续往里加要求：

- `round_2` 就已经不只是在问“怎么看更像强行脱拽”，而是要求先用不超过 4 步给出判断流程，并且每一步都要明确说：这一步在本案里更支持“强行脱拽”、更支持“酒后打闹/扶人”，还是只能保持中性。然后还要求把三种解释放在同一个框架下排顺序：A 强行脱拽、B 酒后熟人打闹、C 醉酒搀扶。
- `round_3` 接着把这三种解释压得更紧：题目要求你必须先抽出一个“最小判别链”，只允许选 4 个事实节点，而且要来自不同时间段；再要求你回到四个固定维度上比较三种解释在每个维度下谁最能吸收这些事实、谁最难吸收这些事实。
- `round_4` 又继续增加反事实要求。它不只让你比较 A/B/C 在四个维度里的解释力，还要求你回答：如果把“第三人经过后男子短暂停手”这条事实拿掉，排序会不会变；如果把“女子挣脱离开并整理衣物”改成“两人并肩离开，女子没有整理衣物”，排序会不会变；并且要比较这两个变化，哪一个更会真正动摇原先的判断链。
- `round_5` 最后一轮继续追问得更细：题目开始要求你指出，四个维度里哪个才是主轴维度、哪个只是校准维度；如果删掉其中一个维度，哪一种解释最先松动；还要求选出一个最容易误导人的特征，先替“打闹”或“搀扶”尽量辩护，再解释这条辩护卡在哪个边界条件上，一旦越过这个边界为什么就会倒向“强行脱拽”。

最终 Qwen 只在 `round_5` 掉到 `0.92`，而且掉分也不是因为它把“强行脱拽”和“醉酒打闹”判反了，而是因为题目继续追问“删掉哪个观察点后影响最大”时，它没有选准那个最关键的观察点。

所以这题更像是在说明：Qwen 对主判断本身并不弱，真正不稳的是当题目继续追问“几个都重要的观察点里，到底谁最重要”时，它不总能分得很准。

### 5. `18485`

- 轨迹：`0.83 -> 1.00 -> 0.99 -> 1.00 -> 1.00 -> 1.00`
- 题长：`61 -> 2378`

这一题最能说明原始 prompt 的一个老问题：喜欢把“固定哪些画面”这种本来可以问得很实的题，越改越像一份层层展开的长报告。

原题只是问：除了抓取伤人和抢枪瞬间，还应重点固定哪些关联画面。  
到 `round_1`，它被改造成按“案发前—案发中—案发后”列出优先固定画面，并说明各类画面的证明作用、缺失争议、不足条件，以及如何补强既遂/未遂判断。

但之后每轮都继续叠加：

- `round_2`：不再满足于按“案发前—案发中—案发后”列画面，而是要求先从里面筛出最关键的 6 类画面并排优先级；每一类都要继续回答 4 件具体的事：这类画面到底证明什么、如果缺了最容易被人解释成什么、单独看为什么还不够、必须再和前后哪类画面接上才能形成完整证明链。除此之外，还单独要求解释：怎样只靠视频画面去反驳“其实是先发生独立伤害，后来才临时起意碰到配枪”这种说法；以及为什么“枪离套了”还不能直接等于既遂。
- `round_3`：又把题目推进到“先排除连续性争议，再判断既遂/未遂”的两层答法。它不仅要列 6 类关键画面，还要求每类画面都固定回答 6 项内容，包括：最小时间片段该截到哪里、最容易和什么别的画面混淆、看不清时结论该降到哪一步。与此同时，又新增四种竞争解释，要求按“最先排除到最后排除”排序，并逐个说明为什么不能把它们当成更优解释。
- `round_4`：继续把题压得更细。它要求把 6 类关键画面再进一步压成“最小充分证明版”，也就是不能只说“这些画面都重要”，而要继续分清：如果只是想排除“先伤人后偶然碰枪”，最少必须保留哪两类画面；如果想从这里再往上走到更支持既遂，还必须再补哪两类；另外还要专门指出，哪些画面看起来很强，其实只是补强，不能代替前面的必要画面。
- `round_5`：最后一轮又在这个基础上继续加码。它要求把全案写成“先排除、后定性、再逆向校验”的三段流程；要求为既遂/未遂的几种不同情形分别指定“最关键的一帧或一小段连续画面”；要求再列出几条“对连续性是必要但不充分”“对既遂有强诱导性但不能单独证明”的信号；还要求给出两个会让结论翻转的关键画面，也就是原本只能到未遂或存疑，但一旦补到某一帧，为什么会突然更支持既遂，或者反过来从看似既遂降回存疑。

这里后面的变化，基本都不是在改“到底还该补拍哪类画面”这个核心问题，而是在继续往题里加：

- 每类画面还要分别说明什么作用；
- 哪些情况只能支持到哪一步；
- 如果画面看不清，结论要往下退到哪里；
- 还要再比较竞争解释和反事实翻转点。

但得分轨迹却是 `0.83 -> 1.00 -> 0.99 -> 1.00 -> 1.00 -> 1.00`。这说明 Qwen 很吃这种题：题目越要求按层说、按步说、把够不够证明分开说，它越容易稳定展开。也就是说，这类变化不是在打它的短板，反而是在把题目改成它相对擅长的答法。

## 原始问题进化 Prompt 的主要问题

1. **题目先被拉长了，但没有先被问尖**
   真正进化样本的平均题长从 `309.5` 到 `1995.0`，但平均得分率始终在 `0.95` 左右，只到最后一轮才略降。

2. **原始 prompt 很容易把一题改成一串任务**
   它常见的改法不是“把一个关键判断问得更准”，而是：
   - 先让模型逐步判断；
   - 再让模型比较几个说法；
   - 再让模型排序；
   - 再让模型补最少证据；
   - 再让模型处理反事实变化。

   这样做会让题目看起来更复杂，但很多时候测出来的只是“能不能跟上长题格式”，不一定是在测真正的专业判断短板。

3. **真正压分的样本太少，而且错点不够集中**
   这批进化题真正形成明显压分的只有 `801` ；其余多数题目仍然高分，说明原始 prompt 没有把样本定向推向“最容易出错的单能力点”。

4. **多轮之后开始出现空转**
   `9891` 在 `round_5` 直接失败，原因是“进化后的问题与原题完全相同”。说明原始 prompt 在多轮链式进化下开始出现“无效改写”现象，进一步说明它没有复杂度和变化质量约束。

## 结论

- 真正进化样本并没有稳定压出 Qwen 的能力边界，反而表现为“题目越来越长，Qwen 仍然高分”。

因此可以直接下结论：**原始问题进化 prompt 更擅长把题目做大，不擅长把问题问准。**

对后续优化的启示是：

1. 问题进化 prompt 必须限制题长和任务数量；
2. 每轮只打一个能力点，而不是把多个推理维度叠在一起；
3. 应优先生成像 `12513`、新版本 `801` 这种“失分原因明确、边界聚焦”的题，而不是 `8638`、`18485` 这种超长但几乎满分的题；
4. 后续分析应继续只关注 `question_evolved=True` 的样本，不要再用整体平均分替代真正的边界诊断。

## 基于当前实验结果判断的 Qwen 能力边界

### 可作为后续优化方向的能力不足

1. **一道题里摆出几个都像缺口的地方时，Qwen 不总能先抓准最卡结论的那个**
   - 代表样本：`801`
   - 具体表现：这题最后不是只让它说“证据还不够”，而是要它先选出“哪一步最证明不了”。更关键的缺口其实是“看不出考生有没有真的收到提示并改了答案”，但 Qwen 更先去谈“动作本身也可能有别的解释”。
   - 为什么这说明是真短板：它不是完全不会分析，也不是整题答反，而是在几个都成立的问题里，没有先挑出最卡结论的那个。
   - 后续优化方向：多做“有两个或三个都像问题点，但必须只选一个最关键缺口”的题。

2. **题干只多加一条新信息时，Qwen 不总能把原来的判断顺序重新排对**
   - 代表样本：`7337`
   - 具体表现：这题后来不断加入换乘、分流、队员反馈和时间窗口变化。Qwen 整体思路能说出来，但有些轮次会把“该先追谁、谁只能做备选、哪条反馈该降权”排得不够稳。
   - 为什么这说明是真短板：它不是不会做现场追踪，而是题干信息一变，原先那套主追/备追顺序不总能跟着调整到位。
   - 后续优化方向：多做“题干只改一条信息，但答案排序必须重新调整”的题。

3. **主判断能做对，但继续追问“几个观察点里谁最关键”时不够稳**
   - 代表样本：`14865`
   - 具体表现：在“强行脱拽”和“醉酒打闹”这题里，Qwen 对大方向判断基本没问题，但到了后面要继续分“删掉哪一个观察点后，结论最先站不住”时，就没选准那个最关键点。
   - 为什么这说明是真短板：它不是一开始就判反了，而是在主判断已经成立的前提下，继续比较“哪个观察点最关键”时不够稳。
   - 后续优化方向：多做“几个判断点都有关，但必须说明哪个最关键、哪个只是辅助”的题。

4. **经常知道“还缺证据”，但不总能说准最少该补哪一条**
   - 代表样本：`801`、一定程度上的 `18485`
   - 具体表现：有些题不是问“你还能补哪些信息”，而是问“如果只能补一条，补哪条最有用”。Qwen 往往知道现在不能直接下结论，但容易把“有帮助的补充信息”和“真能让结论站住的关键事实”混在一起。
   - 为什么这说明是真短板：这不是术语问题，也不是格式问题，而是没抓准“哪一条一补上，结论最可能翻过去”。
   - 后续优化方向：多做“两个补充事实看起来都像有用，但只有一个真的能让结论成立”的题。

### Qwen 已经做得比较好的能力

1. **题目就算越改越长，Qwen 通常也能把答案顺着要求写出来**
   - 代表样本：`18485`、`8638`
   - 具体表现：像 `18485`、`8638` 这种题，后面已经被改成了很长的一串要求，但 Qwen 还是能基本一项项跟上，很少因为题长直接崩掉。
   - 为什么这说明这是稳定优势：它不是偶然一题高分，而是多轮加要求之后仍能持续高分。
   - 后续策略：不要再把“题更长、结构更多”当成主要优化方向。

2. **只要核心还是“现在能不能直接下结论”，Qwen 往往能守住这条线**
   - 代表样本：`801`、`14865`、`12513`
   - 具体表现：它通常知道“看到可疑动作不等于可以直接认定作弊”“看到冲突结果不等于能直接反推前面发生了什么”“有线索不等于已经能写进最终结论”。
   - 为什么这说明这是稳定优势：这些边界在多类题里都能守住，丢分往往发生在更往下一层，比如“哪一个缺口最关键”。
   - 后续策略：不要再把“只能作线索、不能直接定案”这种基础题当成主力测试。

3. **场景里就算有很多人、很多岔路、很多干扰信息，Qwen 通常也能把主思路说清**
   - 代表样本：`7337`
   - 具体表现：像换乘、分流、主追/备追这种现场条件都加进去后，Qwen 通常还是能把追踪思路、备选路线和风险点讲出来。
   - 为什么这说明这是稳定优势：这类题不是一句话就能蒙对，能一直高分，说明它确实能跟住复杂场景。
   - 后续策略：不要只靠加场景复杂度来测，应该在复杂场景里专门追问“最关键的一步是什么”。

4. **题目要求按步骤说、按层说时，Qwen 往往答得比较顺**
   - 代表样本：`18485`
   - 具体表现：像 `18485` 这种题，后面不断要求“分段写”“说明作用”“说明不够在哪里”“说明还缺哪层”，Qwen 通常都能按这个节奏展开。
   - 为什么这说明这是稳定优势：这类题不是只看懂问题就够了，还要能把答案组织出来，而 Qwen 在这方面表现稳定。
   - 后续策略：后面应少做这种“大框架证明题”，多做“只差一个关键判断”的尖题。

### 综合判断

基于 `exp2` 可以认为，Qwen 当前的主要问题不在“长题做不下来”，而在：

- 一道题里有多个问题点时，不总能抓准最关键的那个；
- 题干多加一条信息后，原有判断顺序不总能跟着调整到位；
- 知道“还缺证据”，但不总能说准“最少还缺哪一条”。

而它已经比较稳的地方是：

- 长题和多步骤题的展开；
- 基础边界判断；
- 一般复杂场景下的思路组织；
- 按步骤说明、按层回答这类题。

因此，后续更值得优化和压测的方向应优先集中在：

1. 几个问题点里，哪一个最关键；
2. 最少还缺哪一条关键事实；
3. 几个相近判断之间怎么分主次；
4. 新信息出现后，原来的判断顺序怎么改。

## 未进化低分样本分析

- `13364`: `0.60`

### 总体判断

这些未进化低分样本不能整体当作 Qwen 的能力边界。

更准确地说，它们可以分成三类：

1. **题目真正想问的重点没抓住**：也就是题目其实在问 A，但回答一路答到了 B。
2. **大方向是对的，但漏了评分特别看重的细项**：也就是核心思路没错，只是少了几个指定说法、指定细节或指定例子。
3. **答案写得太满，超出了题目真正要问的范围**：本来只要回答“为什么”，结果一路写到了“怎么做、怎么固定、怎么合规、怎么出庭”。

从这 8 条看，真正最值得当能力边界候选保留的是 `6582`。`5486` 也有一部分这个性质，但里面掺了不少评分口径偏好的影响。其余多数样本，更像是“答得不差，但没刚好答到评分最看重的那些点”。

### 1. `13364`：低分主要来自 rubric 细项，不能直接算能力边界

题目：

- `嫌疑车辆进入山脚等监控盲区后，民警在研判时如何通过进出盲区的时间差来推断其在内部的活动情况？`

得分：

- `6 / 10`

低分点主要是：

- 没有明确说“盲区是封闭区域或有边界区域”；
- 没有使用 `T1/T2/ΔT` 这类变量符号；
- 没有写“原路返回 vs 穿越盲区”的进出方向逻辑；
- 没有提供完整具体数值案例。

但 Qwen 的答案已经覆盖了：

- 理论通行时间基准；
- 实际进出时间差；
- 时间差分级；
