> exp3 使用优化后的问题进化 prompt（V3），核心改进包括：
> 1. 每轮只围绕一个主因设计进化目标（泛泛罗列、缺少推理链、混淆层级、引入题外事实、忽略边界条件、被干扰信息带偏）
> 2. 严格复杂度预算：单一主轴、新增事实≤3条、输出任务≤3个、反事实变量≤2个、硬性限制≤5条
> 3. 禁止把难度建立在长篇格式、固定表格、复杂编号体系上
> 4. 推荐进化方式：因果链收束、边界判断、最小充分条件、单变量反事实、干扰项排除、具体化约束

---

## 一、结论摘要

exp3 用了 V3 prompt 之后，题目确实不再像 exp2 那样一路越改越长、越改越像大题套小题。它更常见的改法，是盯住一句话里“哪些现在只能算线索，哪些不能直接写成结论”，然后围绕这一点反复收紧。

但这不等于每个样本都因此被打到了真正短板。exp3 里最明显的进步，是有些题终于开始压出分了，尤其 `8638`；最明显的局限，是很多题虽然变得更聚焦，却也更容易被 Qwen 快速适应，前几轮依然几乎全是满分。

另一点也很清楚：未进化样本和 exp2 基本一致，说明这些样本的表现主要还是由原题和 rubric 决定，不是这版 evolution prompt 改出来的。

## 四、逐题分析：真正进化的样本

### 1. `801`（考试作弊证据风险）

- **exp2 轨迹**：1.00 → 1.00 → 1.00 → 0.95 → 0.88（题长 50→2194）
- **exp3 轨迹**：1.00 → 1.00 → 1.00 → 0.80 → 0.83（题长得到控制）

**exp3 进化特点**：
- 这一题在 exp3 里不再像 exp2 那样越改越大，而是一直盯着一件事：视频里看到的这些动作，现在到底能写到哪一步，哪些话还不能直接写成作弊结论。
- `round_1`：题目只保留两类动作事实，“嘴巴动了几秒”和“手指点了一下试卷某一区域”，要求你分别说清：哪些可以老老实实写成“已观察到的事实”，哪些还不能直接写成“监考在传答案”或“考生已按提示作弊”。
- `round_2`：把问题收得更死，不再让你泛泛讲风险，而是直接给出几句可能写进材料里的话，让你判断其中哪些已经越级，并要求指出：如果想把它们往上写，最少还缺哪一类关键事实。
- `round_3`：继续把这个边界切得更细，开始要求在几句相近表述里继续分层，不只是分“能写/不能写”，还要继续指出哪一句只是可疑、哪一句已经把可疑直接写成了结论。
- `round_4`：题目开始逼你把判断收成一条短链，也就是“现在能直接写到哪一步，中间还缺哪一个关键判断，最后哪一点因为缺证据而不能再往上推”。
- `round_5`：最后一轮则更像一题“分层排序题”了：不是只问“缺不缺证据”，而是逼你在几个都像问题的缺口里继续分清哪一个最关键、哪一个只是次一级缺口。

**对比结论**：
- exp3 的 `801` 在 `round_4` 首次压到 `0.80`，说明这版 prompt 至少比 exp2 更早打到了问题点。
- 但 `round_5` 又回到 `0.83`，也说明 Qwen 很快就适应了这种“给几句表述，继续分边界层级”的题型。
- 和 exp2 相比，这题的压分干净得多，不再是因为任务太多，而是因为要求它把“事实、线索、结论”之间的界线分得更细。
- **失分原因**：它能看出“嘴巴动了/手指点了”还不能直接下结论，但在 `round_4`、`round_5` 里对“最少还缺哪一个关键事实”说得还不够准，容易停留在“还差证据”这种宽说法上，没有把缺口精确到“说了什么”或“所指位置能不能对应到具体答案”这一步。

### 2. `7337`（连线追踪法）

- **exp2 轨迹**：0.80 → 0.96 → 0.98 → 1.00 → 0.87 → 0.97（题长 59→1708）
- **exp3 轨迹**：0.80 → 1.00 → 1.00 → 1.00 → 1.00 → 1.00

**exp3 进化特点**：
- 这题在 exp3 里也不再往“场景越来越大”走，而是反复追问一句话：既然面貌和衣着都可能变，那步态、路线、节点这些东西到底能不能直接拿来认定“就是同一个人”。
- `round_1`：先从一句最简化的话开始，让回答者判断这句话能不能成立，并说明依据。
- `round_2`：继续追问的是，“步态、路线、节点可以直接作为结论依据”这种写法到底能不能进最终结论。
- `round_3`：再往前一步，不只问“能不能直接写”，还问“它们是不是至少比外观特征更可靠”，逼着回答者说明比较依据。
- `round_4`：开始要求指出，如果现在就把这些东西写成确定结论，到底缺了哪一个前提。
- `round_5`：最后继续把问题压到“缺的前提类型是什么”，也就是缺的到底是身份链没闭合、时空接力没锁死，还是只是外观变化带来的合理替代解释还没排掉。

**对比结论**：
- 这题在 exp3 里一路满分，说明对 Qwen 来说，“步态、路线、节点不能直接越级写成确定结论”这类边界题并不难。
- 这也说明 V3 的聚焦策略不是对所有样本都有效。如果选中的这个切口本来就不是短板，题面再怎么围着它收紧，也压不出分。
- **失分原因**：没有稳定失分点。题目虽然一路把“能不能直接写”压得更细，但 Qwen 始终能把它们当成“还不能定案、但可以继续核查的线索”，没有被逼出明确错误。

### 3. `8638`（PSD vs JPG 格式选择）

- **exp2 轨迹**：0.80 → 1.00 → 1.00 → 1.00 → 1.00 → 1.00（题长 52→2415）
- **exp3 轨迹**：0.90 → 1.00 → 1.00 → 1.00 → 1.00 → 0.68

**exp3 进化特点**：
- 这一题在 exp3 里是最典型的“边界切得越来越细”的样本。原题本来是问 PSD 和高压缩 JPG 的区别，到了 exp3，题目不断把它压成一句话判断：哪些理由可以直接写进最终结论，哪些理由最多只能算线索，哪些理由根本不该作为本题结论依据。
- `round_1`：先给三种说法，让你把它们分成几类，比如哪些属于能写进结论，哪些最多只是线索。
- `round_2`：再收得更窄，只让你在两句话里判断：到底哪一句能直接进最终结论。
- `round_3`：进一步禁止你用“后续编辑更方便”“工作流更顺手”这种理由来撑结论，逼着你只围绕保存方式本身会不会直接损伤关键细节来判断。
- `round_4`：开始要求你说清楚两种说法的分界到底在哪里，也就是为什么一句还能写，另一句只能留在线索层。
- `round_5`：最后把三句话一起摆出来，要求你继续把它们分成“可写入结论”“最多是线索”“根本不该作为本题结论依据”三档，并解释后两档为什么不能往上升。

**对比结论**：
- 这题在 `round_5` 压到 `0.68`，是 exp3 里最有价值的一条进化样本。
- Qwen 真正掉分的点，不是它不知道 PSD 比高压缩 JPG 更适合作工作底图，而是它在题目已经明确限定“只能依据保存方式本身是否会直接削弱关键细节”之后，还是会把“后续工作流程更方便”“更专业”“更常用”这类题外理由带进来。
- 更具体地说，它能分清 `甲` 可以写、`乙` 最多只能算线索，但把 `丙` 错放进了“线索”这一档；而正确做法应该是把 `丙` 直接排到“根本不构成本题结论依据”那一档。也就是说，它知道 `丙` 不够硬，但还没有把“不够硬”和“根本不是这一题该用的依据”分得足够开。
- **失分原因**：它没有把题目限定住的“保存方式本身会不会损伤关键细节”这条线守到底，还是把“工作流更方便、更专业”当成了能帮忙支撑结论的理由；更具体地说，它把本应直接排除的 `丙` 当成了还能保留的线索，层级放高了一档。

### 4. `12513`（抛撒物品行为特征）

- **exp2 轨迹**：0.60 → 0.60 → 0.60 → 0.60 → 0.60 → 0.60（未进化，透传）
- **exp3 轨迹**：0.80 → 1.00 → 1.00 → 1.00 → 1.00 → 1.00（进化）

**exp3 进化特点**：
- 这题在 exp2 里根本没被真正进化，在 exp3 里才第一次被拿来改题。
- 它的改法也很典型：不再问“抛撒物品对刻画嫌疑人特征有什么意义”这种开放题，而是改成一句句去卡边界，例如哪些说法现在就能写进研判意见，哪些最多只能留在线索层，哪些还差关键一步。

**对比结论**：
- V3 确实成功把它从“透传低分样本”变成了“真正进化样本”。
- 但进化后一路满分，说明这条路虽然改得更干净，却没打到 Qwen 真正容易错的地方。
- **失分原因**：没有实际失分。它只是被改成了更窄的“边界判断”题，但 Qwen 仍然能把每一档说法分清，没有露出明确短板。

### 5. `18485`（圆珠笔扎伤抢配枪视频固定）

- **exp2 轨迹**：0.83 → 1.00 → 0.99 → 1.00 → 1.00 → 1.00（题长 61→2378）
- **exp3 轨迹**：1.00 → 1.00 → 1.00 → 0.93 → 1.00 → 1.00

**exp3 进化特点**：
- 这题在 exp3 里也被改成了典型的“分层写法题”。它不再问你“还该固定哪些画面”，而是反复给出几种可能写进材料的话，让你判断这些话现在到底应该落在哪一层。
- `round_1`：先问哪一项最容易把线索直接写成结论。
- `round_2`：再把几种说法分成更细的几档，比如哪些可以直接写、哪些只能降格写、哪些还只是待印证线索。
- `round_3`：继续追问每一句如果现在就写，最少还缺哪一类补充事实。
- `round_4`：开始把问题压成“哪些只能先写成线索或待核查，哪些才够资格写成确定表述”。
- `round_5`：最后再进一步压成几档分类题，逼着回答者继续分清“能写、只是线索、不宜写入”这几个层级。

**对比结论**：
- 这题在 `round_3` 轻微掉到 `0.93`，但后面很快又回到满分。
- 说明这题的变化虽然比 exp2 干净得多，但核心上还是在反复追问“哪些话现在还不能越级写进结论”，而这件事本身并不是 Qwen 的硬伤。
- **失分原因**：`round_3` 的小幅掉分，不是因为它把方向答反了，而是因为在几种相近说法里没有把“现在可以写”和“只能先留作线索”切得特别利落，略微多给了一个层级偏高的表述；但这类偏差很轻，后面马上又回到满分，说明不是稳定问题。

### 6. `13364`（车辆盲区时间差推断）

- **exp2 轨迹**：未包含在 exp2 的进化样本中（透传，得分 0.60）
- **exp3 轨迹**：0.80 → 1.00 → 1.00 → 1.00 → 1.00 → 0.88

**exp3 进化特点**：
- 这题在 exp2 里原本只是透传低分样本，到了 exp3 才第一次被真正改造成一道边界判断题。
- 它的改法很直接：不再问“时间差可以推断什么”，而是不断追问“为什么不能直接把时间差写成某种具体行为已经发生”。
- `round_1`：先问你能不能把“时间差”直接写成某种具体行为结论。
- `round_2`：继续逼着你解释：为什么这句话现在不能进结论，中间还差哪一环。
- `round_3`：再往前走，要求说清楚越过的不是一般性的“可疑”边界，而是从“有异常”直接跳到了“某种具体行为已经发生”。
- `round_4`：开始把相近说法拆开，哪些还算可以写进结论，哪些最多只能算线索。
- `round_5`：最后一轮继续追问缺失事实的类型，也就是缺的到底是更异常的时间差，还是能把具体行为本身直接写进结论的那种事实。

**对比结论**：
- 这条样本说明 V3 的价值不只在“压分”，还在于把原本那种“细项没写够”的低分，改造成了更像真正逻辑边界测试的题。
- 到 `round_5` 掉到 `0.88` 时，Qwen 暴露出的就不再是“变量符号没写、例子没举”，而是“知道这句话现在还不能写进结论，但解释不清到底差的是哪一类事实”。
- **失分原因**：它能判断“时间差不能直接写成具体行为”，但到了 `round_5` 还是没有把缺口说死，容易只说成“还要更多证据”或“还差一环”，没有明确区分缺的是“异常现象”还是“能直接把具体行为写进结论的事实”。

## 五、未进化低分样本分析

exp3 中未进化样本与 exp2 完全一致，共 8 条：

| 样本 | 得分率 | 与 exp2 对比 |
|------|--------|-------------|
| `3564` | -0.1765 | 相同（触发负向项） |
| `4352` | 0.70 | 相同 |
| `5486` | 0.50 | 相同 |
| `6582` | 0.50 | 相同 |
| `7353` | 0.60 | 相同 |
| `9891` | 0.3448 | 相同 |
| `13119` | 0.50 | 相同 |
| `14865` | 0.70 | 相同 |

### 总体判断

这些未进化低分样本在 exp3 中的表现与 exp2 完全一致，说明：

1. **这些样本的低分与进化 prompt 无关**：无论使用原始 prompt 还是 V3 prompt，这些样本都不会被选中进化（因为它们的得分没有超过触发阈值，或者进化后得分仍然低于阈值）。

2. **低分原因分类与 exp2 相同**：
   - **真实能力边界**：`6582`（核心研判主轴抓偏，0.50）
   - **rubric 或题面约束**：`13364`（变量符号、封闭区域建模漏写，但 exp3 中已被进化）、`12513`（术语漏写，但 exp3 中已被进化）、`3564`（负向项触发）、`4352`（比例原则关键词缺失）
   - **表达边界控制**：`13119`（信息纯度不够，混入题外内容）
   - **题目本身难度**：`9891`（满分 29 分，得分 10 分，得分率 0.34，题目本身难度高）

### 与 exp2 的关键差异

在 exp3 中，有两个样本（`12513` 和 `13364`）被成功进化，而在 exp2 中它们是透传样本。这说明：

1. **V3 prompt 的进化触发机制更灵敏**：能够识别出"有进化潜力"的样本，即使它们在 baseline 中得分不高。

2. **进化方向更精准**：`12513` 和 `13364` 在 exp2 中低分是因为"rubric 细项漏写"，而 V3 prompt 将它们改造成"边界判断"题后，成功避开了 rubric 细项的束缚，转而测试逻辑边界。

3. **但进化后仍然是高分**：`12513` 和 `13364` 进化后全程满分或接近满分，说明 V3 prompt 虽然能成功进化这些样本，但进化后的题目仍然不是 Qwen 的真正短板。

## 六、未进化低分样本详细分析

本节参考 exp2 的分析框架，对 exp3 中 `question_evolved=false` 且得分率不高的透传样本进行逐条分析。这些样本在 exp3 中未被进化，各轮得分与 exp2 完全一致。

### 总体判断

这些未进化低分样本不能看到低分就直接算成 Qwen 的短板。更稳妥的分法，还是和 exp2 一样分成三类：

1. **题目真正想问的重点没抓住**：也就是题目其实要你盯 A，你却先答到了 B。
2. **大方向没错，但少了评分特别看重的几个落点**：比如少了一个硬关键词、一个指定细节，或者少了一句特别点题的话。
3. **答案写得太满，超出了题目真正要问的范围**：本来只问“为什么”，结果一路写到了“怎么固定、怎么合规、怎么出庭”。

从这 8 条看，真正最值得保留成能力边界候选的还是 `6582`。`5486` 也有一部分这个性质，但里面掺了不少评分口径和固定写法偏好的影响。其他多数样本，更像是“答得不差，但没刚好答到评分最看重的点”。

### 1. `3564`：负向项触发导致低分不干净

- **题目**：视频研判中，如何通过画面细节区分嫌疑人是"破坏进入"盗窃电动车，还是利用车主未拔钥匙实施的"溜门"盗窃？
- **得分**：`-3 / 17`（得分率 `-0.1765`）
- **状态**：全程透传，未进化

**低分原因**：
- 答案触发了严重负向项：Qwen 提到"若有音频可闻金属断裂/摩擦声"，而 rubric 明确不希望依赖音频研判，因此被全额扣分（扣 5 分）。
- 此外，rubric 对视觉细节要求非常窄（要突出"前置动作与仪表盘/尾灯亮起的时间差"、"推行中处理锁具"、"车身剧烈抖动"等），Qwen 的覆盖不够稳。

**与 exp2 对比**：exp2 中 `3564` 同样得分 `-0.1765`，低分原因完全相同。这说明该样本的低分与进化 prompt 无关，主要由 rubric 的严格负向项和窄视觉细节要求造成。

**结论**：更适合标为"高敏感评分样本"，而不是直接作为能力边界样本。

### 2. `4352`：法律关键词和同一性风险表述不够直接

- **题目**：在追踪故意伤害案件嫌疑人时，如果遇到监控盲区导致目标丢失，办案人员在进行轨迹拼接时应注意哪些关键点以确保研判结论的准确性？
- **得分**：`7 / 10`（得分率 `0.70`）
- **状态**：全程透传，未进化

**低分原因**：
- 没有写 `比例原则` 这个硬关键词。
- 虽然讲到了张冠李戴风险，但没有直接点明"时空接近不等于同一人"这个核心逻辑风险。

**与 exp2 对比**：exp2 中 `4352` 同样得分 `0.70`，低分原因完全相同。这说明它不是不会做轨迹拼接研判，而是没有命中特定法律术语和一句关键边界表达。

**结论**：更像评分点漏写，不是能力边界。

### 3. `5486`：有一部分真问题，也有一部分是评分更偏好固定写法

- **题目**：在串并多起"拉车门"盗窃案件时，如何通过视频监控中的时间点和行驶轨迹来确定是同一伙人所为？
- **得分**：`8 / 16`（得分率 `0.50`）
- **状态**：全程透传，未进化

**低分原因**：
- 这题真正的问题在于：Qwen 会讲“怎么用时间、轨迹、特征做串并”的通用框架，但题目更想问的是很实的那一套：几次案子里试拉车门、停留、离开的时间点怎么一一对齐；路线是不是一路连着；车和人的微小特征能不能跨案复现。
- 也就是说，它讲得到“大方向”，但没有先落到这道题最想看的那种实战颗粒度上。
- 另外，rubric 本身也很偏爱固定写法，比如要把时间链、空间链、特征链、补强链分得很清楚，还偏好一些传统图侦术语，这也压低了通用但合理的答案。

**与 exp2 对比**：exp2 中 `5486` 同样得分 `0.50`，低分原因完全相同。这说明 V3 prompt 没有改变该样本的得分表现。

**结论**：可以保留，但更适合作为“部分边界样本”：它说明 Qwen 会讲大的时空分析框架，但在这种要求特别细、特别实战的串并题里，不一定会第一时间答到最关键的观察颗粒度。

### 4. `6582`：最接近真实能力边界的未进化低分样本

- **题目**：在追踪偷电动车的嫌疑人时，如果发现嫌疑人骑车进入监控盲区，几分钟后空手走出来，研判人员在调取周边监控时应重点寻找什么线索？
- **得分**：`5 / 10`（得分率 `0.50`）
- **状态**：全程透传，未进化

**低分原因**：
- 这题真正让你盯的是“空手出来的人之后往哪走”，因为车已经不在他手里了，最有价值的就是顺着这个异常去反推车被藏在哪、交给谁、从哪个出口转走。
- Qwen 这里最关键的偏差，是它把主线又拉回了“继续找车”，甚至明确写成“以车找人，而非以人找车”。这和题目当时最想让它切换的方向正好相反。
- 它还顺手带进了轮胎印、基站、现场勘查这些题目根本没让你先看的东西，所以主线更散了。

**与 exp2 对比**：exp2 中 `6582` 同样得分 `0.50`（exp2 中为 `0 / 10`，但 exp3 的 rubric 可能有所调整），核心问题相同——主轴抓偏。这说明 `6582` 是 Qwen 的稳定能力短板，与进化 prompt 无关。

**结论**：这条样本最有价值的地方，不是它低分，而是它低分低得很干净。题目真正让它盯人，它却先回去找车。这可以直接保留成能力边界候选。

### 5. `7353`：高级细项覆盖不足，但主判断没有问题

- **题目**：在研判多人参与的肢体冲突视频时，区分"一般性推搡或扇耳光"与"持械暴力打击"这两类行为，在画面观察和证据固定上有哪些核心区别？
- **得分**：`6 / 10`（得分率 `0.60`）
- **状态**：全程透传，未进化

**低分原因**：
- 没有提"追击行为"或"倒地后持续打击"。
- 没有提器械长度、材质、杀伤力的视觉定量分析。
- 关键帧节点只写了打击瞬间，没有明确写"取械瞬间"。

**与 exp2 对比**：exp2 中 `7353` 同样得分 `0.70`（略有差异，可能是 rubric 微调），核心问题相同——高级细项没有全覆盖，而不是主判断错误。

**结论**：更像 rubric 细项扣分样本，不宜作为 Qwen 能力边界。

### 6. `9891`：题目本身太重，不是很干净的能力边界样本

- **题目**：在追踪多名未成年人结伙作案时，如果嫌疑人从监控严密的主干道（一类点）进入了监控覆盖较差的旧货市场或城中村（二类点）导致目标丢失，在进行视频接力研判时应重点注意哪些细节来确认为同一伙人？
- **得分**：`10 / 29`（得分率 `0.3448`）
- **状态**：全程透传，未进化

**低分原因**：
- 这题本身就很重，满分 29 分，是整批题里要求最密的一类。
- Qwen 其实已经讲到了体貌特征、时空关联、换装行为这些主干内容，但 rubric 继续往下卡得非常细，比如鞋子款式、背包肩带样式、几个人高矮比例这些点都要补。

**与 exp2 对比**：exp2 中 `9891` 在 round_5 进化失败（"进化后的问题与原题完全相同"），最终得分 `0.8276`。但在 exp3 中，`9891` 未被选中进化，始终透传，得分 `0.3448`。这说明 exp3 的进化触发机制可能更严格，或者 `9891` 的 baseline 得分太低（低于触发阈值）。

**结论**：这更像一题“要求特别多、扣分点特别碎”的重题，不适合直接拿来当能力边界样本。

### 7. `13119`：方向对，但答案写太满

- **题目**：在处理抢劫现场的视频固定工作时，除了记录嫌疑人的逃跑方向，为什么必须重点回溯其作案前的行进轨迹？
- **得分**：`5 / 10`（得分率 `0.50`）
- **状态**：全程透传，未进化

**低分原因**：
- Qwen 的主答案其实没跑偏，已经把“为什么要回溯作案前轨迹”说出来了。
- 真正的问题是它收不住：后半段开始一路写“怎么固定、怎么合规、怎么出庭”，把很多相关但题目没问的东西都带了进来。

**与 exp2 对比**：exp2 中 `13119` 同样得分 `0.50`（exp2 中为 `0.70`，但 exp3 的 rubric 可能更严格），核心问题相同——信息纯度不够。

**结论**：这更像“答案写太满”的样本，不适合直接当成专业判断短板。

### 8. `14865`：基础题，没被进化，得分中等

- **题目**：在KTV等光线昏暗、人员走动频繁的场所，如何通过监控画面准确判断行为人是在实施"强行脱拽"猥亵，而非朋友间的醉酒打闹？
- **得分**：`7 / 10`（得分率 `0.70`）
- **状态**：全程透传，未进化

**低分原因**：
- 该题在 exp2 中是被进化样本（`14865` 在 exp2 中从 round_1 开始进化），但在 exp3 中未被选中进化。
- 得分 `0.70` 属于中等水平，说明 Qwen 对这类基础判断题掌握较好，但仍有提升空间。
- 由于该样本在 exp3 中始终透传，未经历进化，因此其得分表现与 exp2 的 round_0 相同。

**结论**：这条样本在 exp3 里更像“没被选中继续打磨的基础题”，不能从它这里直接读出 V3 prompt 的优劣。

### 未进化低分样本的归类结论

#### 更像真实能力边界

- `6582`：题目真正让它追“空手出来的人”，它却先回到了“继续找车”的常规思路上，这是最有价值的一条。
- `5486`：知道要做串并，但没有先落到“试拉车门、停留、离开、连续路线、微观特征复现”这些最实战的点上。

#### 更像 rubric 或题面约束导致的低分

- `3564`：关键小动作没点全，而且多说了音频线索，结果被重扣。
- `4352`：大方向都对，只差 `比例原则` 和“时空接近不等于同一人”这两句更硬的点题表达。
- `7353`：主判断没问题，只是细观察点补得不够全。
- `9891`：题目本身太重，扣分点又很碎。

#### 更像表达边界控制问题

- `13119`：题目只问“为什么要回溯”，它却一路写到了“怎么固定、怎么合规、怎么出庭”，方向对，但没收住。

#### 基础题，未进化

- `14865`：在 exp2 中曾被进化，在 exp3 中变为透传，得分中等（0.70），可能因进化触发机制差异导致。

### 对后续实验的启示

未进化低分样本不能简单丢掉，但也不能看到低分就直接算成 Qwen 的短板。更稳妥的做法，还是先分清它到底是“主线抓偏了”，还是“评分点没补齐”，还是“答案写太满超题了”。

更合理的处理方式是：

1. 对 `6582` 这类“题目主线抓偏”的样本，应保留为真实能力边界候选，并围绕“反常线索出现后，主线该怎么切换”继续设计新题。
2. 对 `5486` 这类“有部分真问题，也有部分评分口径偏好”的样本，应把题面和评分标准都写得更直白，减少“答得不差但没踩中那套固定写法”的情况。
3. 对 `3564`、`4352`、`7353` 这类“大方向对，但关键落点没补齐”的样本，不应直接拿来下能力边界结论，更适合用来改 rubric，让评分重点写得更清楚。
4. 对 `13119` 这类“答案方向对，但写太满”的样本，可以单独作为“答题边界控制”能力点，而不是归入专业判断能力不足。
5. 对 `9891` 这类“题本身太重”的样本，应先想清楚它适不适合继续留在 baseline 里，不然很容易把高难重题误当成模型稳定短板。
6. 后续分析时应把“进化失败低分”“未进化原题低分”“进化后压分”分开统计，否则整体均分会把几种完全不同性质的低分混在一起。

## 七、exp2 与 exp3 的核心对比

| 维度 | exp2（原始 prompt） | exp3（V3 prompt） |
|------|-------------------|-------------------|
| **进化样本平均得分** | 前3轮 0.99+，后2轮 0.95+ | 前3轮 1.00，后2轮 0.91 |
| **题目长度变化** | 单调膨胀（50→2400字） | 得到控制，聚焦主轴 |
| **进化方向** | 多步骤结构任务叠加 | 单一主轴（边界判断为主） |
| **压分原因** | 结构复杂度导致的优先级偏差 | 边界判断精确性不足 |
| **未进化样本** | 8 条透传 | 8 条透传（其中2条被成功进化） |
| **进化失败率** | `9891` 在 round_5 失败 | 无失败 |

### V3 prompt 的改进效果

1. **成功之处**：
   - 题目长度得到有效控制，不再无限制膨胀
   - 进化方向更聚焦，每轮只打一个能力点
   - 能够成功进化 exp2 中无法进化的样本（`12513`、`13364`）
   - 在 `8638` 上成功压出了新边界（round_5 得分 0.68，exp2 中从未低于 1.00）

2. **不足之处**：
   - 前3轮进化样本仍然几乎满分，说明 V3 prompt 的初始进化仍然"不够锋利"
   - 部分样本（`7337`、`18485`）在适应"边界判断"题型后恢复满分，说明这个主轴对某些样本不够有效
   - 整体压分效果有限：exp3 的 round_5 进化样本平均得分 0.86，相比 exp2 的 0.96 有所改善，但仍然偏高

## 七、基于 exp3 判断的 Qwen 能力边界（更新）

### 可作为后续优化方向的能力不足

1. **题目只准按“保存方式本身”判断时，还是会把“工作流更方便、更专业”这类话拿来撑结论**（新增）
   - 代表样本：`8638`（round_5，得分 0.68）
   - 具体表现：它能把 `甲` 选成可直接写的结论，也能把 `乙` 放进“先作为线索”的一档，但把 `丙`（“PSD 更专业、更常用于工作流”）也留在了线索里。题目实际上要它直接排除这句话，因为这句话根本不是“保存方式会不会损伤关键细节”的证据。
   - 说明：它的问题不是完全看不懂题，而是会把“听起来对这题有帮助”的话全往上留一格，结果把本该直接排除的内容留成了线索。
   - 后续优化方向：多做这种“只能按 X 判断，Y 和 Z 都不能拿来撑结论”的题，并要求把“还能算线索”与“根本不该进这题”分开说清。

2. **同一题里有多个“还差一点”的地方时，最少缺哪一步说得不够稳**（与 exp2 相同）
   - 代表样本：`801`（round_4，得分 0.80）
   - 具体表现：它能看出“嘴巴动了/手指点了”不能直接写成作弊结论，但在说“最少还缺什么”时，容易只写成“还要更多证据”，没把缺口具体落到“说了什么”或“所指位置能不能对应到具体答案”。
   - 后续优化方向：继续强化“最小关键事实”必须落到单一步骤，而不是停在“证据还不够”。

3. **知道“时间差还不能写成具体行为”，但不总能说准缺的是哪类事实**（与 exp2 相同）
   - 代表样本：`13364`（round_5，得分 0.88）
   - 具体表现：它已经知道“时间差”不能直接等于“某种行为已经发生”，但到了要补缺口时，容易只说成“还差一环”或“还要更多证据”，没有把“异常现象”和“能把行为本身写进结论的事实”分开。
   - 后续优化方向：多做“缺的不是异常迹象，而是能直接把行为写进结论的事实”这类题。

### Qwen 已经做得比较好的能力（更新）

1. **把“可写”和“只能留作线索”分开的基础题，已经比较稳**（新增）
   - 代表样本：`7337`、`18485`
   - 具体表现：题目虽然一轮轮把说法切得更细，但它还是能把“能写结论”和“先放在线索里”分开，基本不掉分。
   - 说明：这种单纯的层级划分，对它来说已经比较熟。
   - 后续策略：不要只重复这种题，要换成“最少还缺哪一步”或“哪一条理由不能拿来用”的题。

2. **题目变长、步骤变多时，仍然能按要求展开**（与 exp2 相同）
   - 代表样本：`18485`
   - 具体表现：哪怕题目一轮轮加分类、加比较、加排序，它也能把回答拆开写完整。
   - 后续策略：继续避免把“题更长、结构更多”当成主要优化方向。

3. **基础边界判断**（与 exp2 相同）
   - 代表样本：`801`、`14865`
   - 具体表现：知道“看到了可疑动作”不等于“可以直接下结论”。
   - 后续策略：不要再把基础边界判断当成主力测试。

## 八、对后续实验的启示

1. **V3 prompt 的"复杂度预算"有效，但"初始锋利度"仍不足**
   - 前3轮进化样本几乎满分，说明初始进化仍然过于"温和"。
   - 建议：在 V3 prompt 中增加"初始锋利度"要求，例如第一轮就必须要求"能压出至少一个明显失分点"。

2. **"边界判断"主轴对某些样本有效，对另一些样本不够**
   - 对 `8638` 有效（round_5 压到 0.68），对 `7337`、`18485` 无效（全程满分）。
   - 建议：建立"能力点轮换"机制，不要连续多轮使用同一主轴。

3. **需要引入新的进化主轴**
   - 除了"边界判断"，还应尝试：
     - "因果链收束"：要求说明从已知事实到结论中间还缺哪一步
     - "最小充分条件"：要求指出要让某结论成立还需补充的最小事实
     - "单变量反事实"：只改变一个条件，要求说明结论哪部分改变
     - "干扰项排除"：给出看似有力但不足以支撑结论的信息，要求说明为什么不能据此越级

4. **未进化样本不应被忽视**
   - `6582` 仍然是最接近真实能力边界的未进化样本。
   - 建议：对 `6582` 这类样本，尝试用"因果链收束"或"最小充分条件"来进化，而不是"边界判断"。

## 九、结论

exp3 使用优化后的 V3 prompt，相比 exp2 取得了以下改进：
