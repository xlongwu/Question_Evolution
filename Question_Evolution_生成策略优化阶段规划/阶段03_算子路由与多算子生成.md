# 阶段03：算子路由与多算子生成

阶段编号：阶段03

阶段名称：算子路由与多算子生成

阶段目标：用 operator 体系替代统一 prompt，让不同样本走不同进化策略。

阶段范围：实现 `operator_router.py`，新增 9 个 operator prompt，并改造 `question_evolution.py` 读取 `evolution_action` 与 `operator_route`。

主要任务：
- 实现规则路由与 memory bank 优先级调整。
- 支持 O1-O9。
- `question_evolution.py` 按 operator 选择 prompt，写入 `operator_used`、`ability_axis`、`boundary_hypothesis`、`expected_qwen_failure`、`expected_evaluation_focus`。

涉及模块/文件范围：`operator_router.py`、`question_evolution.py`、`prompts/router_prompt.py`、`prompts/operators/*.py`、`schemas/evolution_result.schema.json`。

前置依赖：阶段02输出稳定；memory bank 可为空但读取逻辑要兼容不存在文件。

完成标准：同一输入中不同 `candidate_overscore_cause` 能路由到不同 operator；透传样本不触发 LLM 进化。

测试与验证方式：mock route 输入；验证 O1/O2/O4/O8/O9 的代表路径；验证上一轮 O1 满分时禁止继续 O1。

风险与注意事项：避免 operator prompt 重新变成长篇万能 prompt；不要反复使用“最小关键事实”换壳。

预计交付物：router、operator prompt 集、改造后的 question evolution 入口。
