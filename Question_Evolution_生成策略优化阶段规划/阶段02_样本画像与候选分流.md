# 阶段02：样本画像与候选分流

阶段编号：阶段02

阶段名称：样本画像与候选分流

阶段目标：把“高分即进化”改为“先诊断再决定进化、重构、透传或停止”。

阶段范围：实现 `profile_samples.py`、`select_evolution_candidates.py` 和 `prompts/profile_prompt.py`。

主要任务：
- 生成 `sample_profile` 与 `overscore_diagnosis`。
- 输出 `evolution_action`，取值为 `evolve_high_score_overscore`、`reconstruct_low_score_boundary`、`pass_through_or_scoring_noise`、`stop_evolution`。
- profile 阶段只诊断，不推荐 operator。

涉及模块/文件范围：`profile_samples.py`、`select_evolution_candidates.py`、`prompts/profile_prompt.py`、`schemas/sample_profile.schema.json`、`schemas/overscore_diagnosis.schema.json`。

前置依赖：阶段01字段合约完成；需要复用现有 `RotatingAPIClient` 或抽出公共 API 客户端。

完成标准：输入 `round_N/scored.jsonl` 后能输出 `profiled_candidates.jsonl`，且每条样本都有诊断字段和明确动作。

测试与验证方式：mock LLM 返回；覆盖高分虚高、低分真实边界、评分噪声、停止进化四类样本。

风险与注意事项：诊断 prompt 不能同时推荐算子，否则会和 router 冲突；低分样本不能无条件重构。

预计交付物：画像脚本、候选分流脚本、profile prompt、单元测试 fixture。
