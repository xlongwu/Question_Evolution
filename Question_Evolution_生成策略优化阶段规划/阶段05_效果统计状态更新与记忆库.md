# 阶段05：效果统计状态更新与记忆库

阶段编号：阶段05

阶段名称：效果统计状态更新与记忆库

阶段目标：把每轮成功、失败、无效生成沉淀为下一轮路由依据。

阶段范围：实现 `analyze_evolution_effect.py`、`update_sample_state.py` 和三类 memory bank。

主要任务：
- 统计 `score_rate_before/after`、`delta_score_rate`、题长、是否满分、轻量边界命中、置信度、人工复核标记。
- 更新 `evolution_state`。
- 写入 `operator_memory_bank.jsonl`、`failure_memory_bank.jsonl`、`invalid_generation_cases.jsonl`。

涉及模块/文件范围：`analyze_evolution_effect.py`、`update_sample_state.py`、`memory/*.jsonl`、`schemas/effect_analysis.schema.json`、`schemas/evolution_state.schema.json`。

前置依赖：阶段04产生完整 metadata；标准闭环已生成新一轮 `scored.jsonl`。

完成标准：每轮评分后可输出效果分析和下一轮状态；稳定降分样本能停止，连续满分样本能换算子。

测试与验证方式：用前后两轮 scored fixture 验证降分、满分、无效、需人工复核四类状态。

风险与注意事项：轻量命中不能读取 rubric item，也不能做 judge agreement；低置信结果不能直接沉淀为强成功经验。

预计交付物：效果分析脚本、状态更新脚本、memory bank 写入与读取逻辑。
