## 本次实验结论

`exp4` 相比 `exp3` 的最大变化是把很多题从“边界分层”改造成了“最小关键事实 / 最小跳步 / 最小充分条件”型问题。

结果：

1. 它在前两轮就更早压中了部分样本，尤其是 `801` 和 `13364`；
2. 它也更容易塌缩成另一种单一模板，导致不少题虽然更短、更聚焦，但后续几轮没有继续形成区分。

- 它在“更早命中一部分真实短板”上优于 `exp3`；
- 它在“多轮稳定维持区分度”和“题型多样性”上弱于 `exp3`；
- 它继续保持了对题目复杂度的有效控制；
- 但它还没有稳定、高效地把题目推进到 Qwen 的真正能力边界上。

## 总体结果

### 全量样本平均分

- `round_0`: `0.6589`
- `round_1`: `0.6444`
- `round_2`: `0.5672`
- `round_3`: `0.5815`
- `round_4`: `0.5815`
- `round_5`: `0.5475`

从全量均分看，`exp4` 的压分比 `exp3` 更早出现：`round_2` 就已经降到 `0.5672`。  
但这个均分不能直接等价于“后几轮越来越强”，因为它混合了提前退出与留存样本两个因素。

### 真正进化样本

- `round_1`: 7 条进化样本，平均得分率 `0.8816`，平均题长 `320.6`，满分样本数 `5`
- `round_2`: 6 条进化样本，平均得分率 `0.7896`，平均题长 `346.0`，满分样本数 `4`
- `round_3`: 5 条进化样本，平均得分率 `1.0000`，平均题长 `309.6`，满分样本数 `5`
- `round_4`: 5 条进化样本，平均得分率 `1.0000`，平均题长 `344.2`，满分样本数 `5`

### 与 `exp3` 的对照

- `exp3 round_1`: 6 条进化样本，平均得分率 `1.0000`，平均题长 `261.5`
- `exp3 round_2`: 6 条进化样本，平均得分率 `1.0000`，平均题长 `392.8`
- `exp3 round_3`: 6 条进化样本，平均得分率 `0.9881`，平均题长 `441.2`
- `exp3 round_4`: 6 条进化样本，平均得分率 `0.9667`，平均题长 `489.0`

这个结果说明：

1. `exp4` 的题长控制依然明显优于 `exp3`。到 `round_5` 平均只有约 `356` 字，而 `exp3` 已经到 `566` 字。
2. `exp4` 的压分来得更早。`round_1` 和 `round_2` 就已经对一部分样本形成明显区分，而 `exp3` 要到 `round_4/5` 才逐渐出现。
3. `exp4` 的压分不稳定。`round_3` 和 `round_4` 的进化样本均分回到 `1.0000`，不是因为题变得更强，而是因为真正难压的样本在前两轮已经退出，剩下的是更容易被 Qwen 稳定吃满的幸存样本。
4. `exp4` 没有再出现 `exp3` 那种“题目一路膨胀却仍然压不住”的问题；它更常见的问题变成了“题目很短、主轴很清楚，但不够尖”。

## 对进化样本的逐题分析

### 1. `801`：本次最成功的共享样本之一

轨迹：

- `1.00 -> 0.82 -> 0.54`

题长：

- `50 -> 351`

原题只是问：仅凭视频中嘴巴动了或手指点了一下试卷，在证据固定上有哪些风险点。  
`exp3` 对它的处理更偏向“哪些内容最多只能作线索、哪些不能直接写入结论”的边界分层。  
而 `exp4` 在 `round_1` 改成了：如果要把“存在作弊答案传递”正式写进处理决定，还缺哪些最小必要事实。

这一步强迫模型把完整证据链拆成独立断点。

`round_2` 进一步把题目推进成更窄的候选项判断：在几条补强建议中，哪一项并不是最小必要条件，以及为什么它已经被其他条件吸收，或者即使缺它也仍可成立。

Qwen 在这个样本上的失分具有代表性：

- `round_1` 时，暴露出会漏掉“考生是否实际接收并使用”这一关键环节；
- `round_2` 时，它进一步没有分清哪一条是独立必要条件、哪一条其实已被别的条件吸收。

所以，这个样本说明：`exp4` 在 `801` 上不是靠题目加长或结构加重来压分，而是靠**把“证据不够”收束成“哪一个断点才是真正独立缺口”**来形成有效边界。  

### 2. `13364`：压分很早，但题面还有一点可收紧空间

轨迹：

- `0.80 -> 0.35`

题长：

- `46 -> 239`

这一题和 `801` 一样，也是 `exp4` 早压分的代表。

原题是问：嫌疑车辆进入山脚监控盲区后，如何通过进出盲区的时间差推断其在内部的活动情况。  
`exp3` 对它的推进方式，主要是不断收束“时间差最多能支持到哪一层”，一直到 `round_5` 才开始出现明显压分。

`exp4` 的做法则更直接。`round_1` 就把题具体化成：

- `19:10` 进入；
- `19:19` 离开；
- 正常通行时间 `8-10` 分钟；
- 盲区内只有一条可通车主路；
- 然后只问两点：结论成不成立；如果不成立，最少还缺哪一个足以排除“单纯通过”的关键事实。

Qwen 在这里的表现：

- 它知道 `9` 分钟仍落在正常区间，不能直接推出会合或装卸；
- 但它没有说准“最少还缺哪一条”，而是把答案偏向了“是否静止停车”这种更表面但不够决定性的事实。

它能判断“现有材料还不够”，但不一定能精确指出“最小关键缺口到底是哪一个”。

所以，这个样本的价值在于：`exp4` 快速找到了 Qwen 的一个真实短板。  
不过也存在一个问题：当题目追问“最小关键事实”时，题面和 rubric 最好更明确地锁死究竟在补哪一个子判断，否则会留下少量可争辩空间。

### 3. `14865`：新增样本里最值得保留的一题

轨迹：

- `1.00 -> 1.00 -> 1.00 -> 1.00 -> 0.63`

题长：

- `72 -> 422`

这一题在 `exp3` 中没有真正进入共享进化池，所以不能像 `801`、`13364` 那样做逐轮严格对照；但在 `exp4` 内部，它是最有价值的新增样本之一。

原题讨论的是：在 KTV 等光线昏暗、人员走动频繁的场所，如何通过监控判断是“强行脱拽”猥亵，而不是朋友间醉酒打闹。

它前几轮的变化路径很能说明 `exp4` 的思路：

- `round_1` 先问两项最小缺失事实；
- `round_2` 改成只指出最关键的一处推理跳步；
- `round_3` 进一步要求从候选跳步中指出真正错误的一步；
- `round_4` 开始逼模型识别结论里偷偷引入的题干外前提；
- `round_5` 最终只追一个点：从现有监控事实到“强行脱拽式猥亵”之间，最关键但缺失的一步事实是什么，而且理由必须同时扣住“接触部位看不清”和“后续画面缺失”。

Qwen 到 `round_5` 才真正掉分，失分方式也很有价值：

- 它把注意力放在“衣物是否被强行剥离或位移”这一更显眼的动作层；
- 但漏掉了“猥亵”定性真正需要的性指向接触要素。

当结论同时包含“动作发生了”和“行为性质成立了”两层门槛时，它容易先抓住更显眼的那一层，而漏掉真正决定定性的那一步。

因此，这个样本说明：`exp4` 不只擅长压“缺最小事实”，它也开始触到一种更高价值的边界，即**结论中双门槛的主次识别错误**。  
这是后续应该重点保留并继续扩展的方向。

### 4. `18485`：更早压到一点，但没有形成稳定边界

轨迹：

- `1.00 -> 1.00 -> 0.80 -> 1.00 -> 1.00 -> 1.00`

题长：

- `61 -> 338`

这一题的原始问题是：针对嫌疑人使用圆珠笔扎伤他人颈部并抢夺配枪的案件，在固定视频证据时，除了抓取伤人和抢枪的瞬间，还需要重点固定哪些关联画面。

`exp3` 在这一题上虽然结构控制更好，但整体还是偏容易，只有 `round_4` 短暂掉到 `0.93`。  
`exp4` 则在前两轮尝试把它收束成“连续行为链”问题：

- `round_1` 问：若要证明伤害与抢枪是同一连续行为链，还缺哪几类关联画面；
- `round_2` 更进一步，直接问：最少还必须补出哪一类中间画面，才能把两段动作写成同一连续行为链；
- 后几轮则继续围绕“从碰到枪套/枪柄跳到未中断同一夺枪动作”这一中间缺口收束。

Qwen 在 `round_2` 的确掉到 `0.80`，说明它会把“动作过渡”说得比较泛，而没有直接锁定“配枪控制转移/接触连续性”这一更精确的关键点。  
但随着后面问题继续明化，它很快又回到满分。

因此，这一题说明：`exp4` 在这里压到的更像是“最后那一词没踩准”的轻度边界，而不是深层稳定短板。  
它是有一定价值的样本，但还不足以算本轮的核心收获。

### 5. `12513`：比 `exp3` 稍有进步，但还不够尖

轨迹：

- `0.80 -> 1.00 -> 1.00 -> 1.00 -> 1.00 -> 0.90`

题长：

- `50 -> 367`

这一题原本问的是：嫌疑人抛撒物品的行为除了干扰现场外，对于刻画嫌疑人特征还有什么实际意义。

`exp3` 在这题上的主要问题，是虽然始终把主轴收束得较干净，但全程没有形成区分。  
`exp4` 则尝试把它逐步推进成“预设性抛撒习惯”或“事先预置纸张”这样的更硬边界：

- `round_1` 问：若要写成“具有较稳定的预设性抛撒习惯”，还缺哪条最关键补充事实；
- `round_2` 进一步压成：是否能写成“纸张事先已分别预置在两处对应口袋内待抛”；
- `round_3` 到 `round_5` 持续围绕“从能看见取纸到能否写成事先预置”这一时间边界收束。

最终 Qwen 在 `round_5` 掉到 `0.90`，说明这题相比 `exp3` 确实有进步。  
但它暴露出的失分仍然偏轻：

- 它知道还要补“不是日常随身携带或临时塞入”；
- 但没有把“排除逃离过程中再放入口袋”这个更细的时间边界说准。

方向是对的，只是最后一级不够稳。  
所以，这题可以保留为轻度有效样本，但它还不是那种真正强压分的边界题。

### 6. `8638`：相对 `exp3` 的明显退步样本

轨迹：

- `0.90 -> 1.00 -> 1.00 -> 1.00 -> 1.00 -> 1.00`

题长：

- `52 -> 324`

这一题是 `exp4` 最值得警惕的负样本之一。

在 `exp3` 中，它其实是最成功的样本之一：  
通过把“支持 B 的说法”拆成“可写结论 / 只能作线索 / 根本不能作为依据”三层，`round_5` 清楚压出了 Qwen 的细层级判断失误，最终掉到 `0.68`。

但 `exp4` 把它改走了另一条路线：

- 不再做三层边界拆分；
- 转而反复围绕“这句话成立还至少缺什么前提”“哪里偷偷引入了题干外假设”“即便补了这个前提为什么仍不能成立”这些隐藏前提识别题去推进。

结果是：

- 题面更短了；
- 主轴也更聚焦了；
- 但 `Qwen` 从 `round_1` 到 `round_5` 全程 `1.00`。

这说明问题根本不在长度，而在题型。  
`exp4` 把一个本来已经有效命中 Qwen 细层级边界的题，替换成了它已经很熟的“识别隐藏前提”题型，结果反而完全失去区分度。

所以，这个样本得出的结论很清楚：`exp4` 并不是所有方向都比 `exp3` 更强。  
如果把有效的细层级分界题换成模型已经熟练掌握的前提识别题，题再短、再干净，也会是退步。

### 7. `7337`：收束得很干净，但从头到尾没有打到边界

轨迹：

- `0.80 -> 1.00 -> 1.00 -> 1.00 -> 1.00 -> 1.00`

题长：

- `59 -> 331`

这一题很能说明 `exp4` 的另一个结构性问题：题型塌缩。

原题只是一般性地问：利用连线追踪法确定嫌疑目标时，除了面貌和衣着，还要结合哪些因素综合分析，以防跟丢目标。  
`exp4` 没有把它扩展成复杂追踪流程题，而是一直围绕“同一目标连续性”这个单一缺口去打磨：

- 从“哪里少了关键中间推理”；
- 到“还缺哪一个最小充分事实”；
- 再到“已有线索很多为什么仍不能推出基本不断线”；
- 后几轮本质上都还是同一个问题。

这在结构控制上当然是成功的：题面一直很短，主轴一直很单。  
但它的问题同样明显：五轮都在换“最小事实/最小环节/最小充分条件”的说法，本质没有真正换题型。

Qwen 在这里全程满分，说明它对这种“指出连续性缺口”的问题已经非常稳。  
因此，这个样本很好地说明：**收束得很干净，并不自动等于有区分度；如果 prompt 只是在同一类缺口问法上换壳，最终只会形成题型塌缩。**

## 对 `exp4` 的总体判断

### 优点

1. 继续有效控制了题目长度和结构复杂度，没有回到旧版那种大幅膨胀的长题模式。
2. 更早压到了一部分真实短板，尤其是 `801` 和 `13364`，说明“最小关键事实 / 最小跳步”这一方向是有价值的。
3. 新增样本 `14865` 暴露出了一类比单纯“证据不够”更有价值的边界：结论含双门槛时，模型容易抓住显眼动作层，漏掉真正决定定性的那一层。

### 缺点

1. 很容易塌缩成“还缺哪一个最小事实”这一种单一模板，导致多轮之间只是换壳，不是真正换题型。
2. 对一部分样本，`exp4` 虽然更早压分，但后续推进能力不足，难以持续维持区分度。
3. 某些在 `exp3` 中已经证明有效的细层级分界题，到了 `exp4` 反而被换成了 Qwen 已经熟练掌握的题型，典型就是 `8638`。
4. `round_3` 和 `round_4` 的高均分说明：这版 prompt 还没有形成稳定的“后续继续推进”机制，早压中的题容易提前退出，留下的题又容易被吃满。

## 结论

`exp4` 是一次方向上有价值、但形态上还不稳定的改进。

- 它解决了 `exp3` “压分来得太慢”的问题；
- 它证明了“最小关键事实 / 最小跳步”确实能更早命中一部分能力短板；
- 它也暴露出：如果这一类问法被过度复用，就会迅速塌缩成新的模板，导致后续轮次失去区分能力。

更具体地说，`exp4` 不是把题目从“复杂”变成“更强”，而是把题目从“层级边界题”转成了“最小缺口题”。  
这种转向对一部分样本是成功的，但它还不足以稳定覆盖更广的能力边界。

因此，下一步优化方向不是回退到 `exp3`，也不是继续单纯强化“最小关键事实”，而是：

1. 保留 `exp4` 在早压分上的优势；
2. 保留 `exp3` 在细层级分界题上的有效样本机制；
3. 强制避免同一样本连续多轮都只问“还缺哪一个最小事实”；
4. 在题型上增加：
   - 候选缺口二选一
   - 子判断定位
   - 近似项分层
   - 双门槛结论拆分
5. 让题目的核心错误尽量集中、可归因，但不要再让这种“集中”退化成固定模板。

换句话说，`exp4` 已经说明：

- 题目不需要长，依然可以有效压分；
- 但“更短、更聚焦”本身不是充分条件；
- 真正关键的是：能不能持续把题目推进到**模型最容易在一个锋利判断点上犯错**的位置。

## 当前版本 prompt 的主要问题

当前这版问题进化 prompt 相比 `exp3` 已经明显改善了两个问题：

1. 题目复杂度没有继续膨胀；
2. 压分出现得更早。

但从本次实验结果看，它仍然存在以下结构性问题：

1. 过度依赖“最小关键事实 / 最小前提 / 最小跳步”这一类题型，导致不少样本后续只是问法换壳。
2. 没有在题目层面强制区分“当前到底在考结论里的哪一个子判断”，这会导致像 `14865`、`13364` 这类题在题面与 rubric 之间仍留下少量暗约定。
3. 对已经满分通过的样本，缺少明确的换题型机制，因此很容易在后续轮次围绕同一类缺口反复打转。
4. 它已经证明“早压分”是可以做到的，但还没有证明“多轮稳定推进”也能自然出现。

## 优化后 prompt

下面按照上一版分析文档的命名规则，给出 `QUESTION_EVOLUTION_PROMPT_TEMPLATE_V5`。  
这一版不是回到 `exp3` 的边界分层题，也不是继续沿用 `exp4` 的“最小关键事实”模板，而是把两者结合起来，并显式约束：

- 先拆目标结论，再决定本轮到底考哪一层；
- 如果有两个都说得通的最小缺口，就不要再做填空式唯一答案；
- 如果上一轮已经在问“最小关键事实”，下一轮必须换题型。

```python
QUESTION_EVOLUTION_PROMPT_TEMPLATE_V5 = """
# 角色
你是一位负责“能力边界定向挖掘”的评测题目设计专家。你的任务不是把题目机械改难，也不是把题目改成长篇结构题，而是要把原题升级成一道**单主轴、可回答、可稳定评分、并且能让弱模型在一个可归因的错误点上失分**的新题。

# 核心目标
请分析原题、参考答案、候选答案和现有评分标准，生成一道升级后的新题，使其满足：
1. 仍然考查原题的核心领域、核心事实和核心能力，不改变题目主题；
2. 新题的主要失分点尽量集中在 1 个核心错误上；
3. 该错误最好不是“不会写长答案”或“没跟上复杂格式”，而是：
   - 说不准最少到底缺哪一条；
   - 分不清两个都像有用的补充事实谁才更关键；
   - 抓住了显眼动作，但漏掉真正决定定性的那一层；
   - 把“高度像”误当成“已经能写成结论”；
4. 后续 judge 能根据 rubric 稳定判断对错，而不是靠篇幅、术语密度或格式完整性打分。

# 第零步：先拆结论，再决定考哪一层
在内部先做这一步，但不要把中间推理全部输出：
1. 如果原题或候选答案里有一个目标结论，请先把它拆成 1-3 个子判断。
   例如：
   - “实施了强行脱拽式猥亵”可拆成“发生了特定拉扯/脱拽动作”与“该动作已达到猥亵定性所需的性质层”；
   - “两段画面属于同一连续夺枪动作”可拆成“起点动作发生”“中间连续性未断”“终点控制完成”。
2. 本轮只能选择其中 **1 个最值得压测的子判断** 作为主轴，不得同时考多个子判断。
3. 如果不先拆结论就会出现“最小缺口有两个都说得通”的情况，则必须先改题型，不得直接做填空式“还缺哪一条”。

# 第一步：诊断候选答案的虚高主因
你必须先在内部从以下主因中选 1 个最主要的：
- A. 泛泛罗列，缺少本题具体事实绑定；
- B. 只说结论，缺少必要中间推理链；
- C. 混淆层级，把线索/可能/待核查升级成可写结论；
- D. 引入题干外事实、常识、经验或新前提；
- E. 知道“不够”，但说不准哪一条才是最小关键事实；
- F. 结论里有两个门槛时，只抓住更显眼的那个，漏掉真正决定定性的那一个；
- G. 被干扰信息带偏，没有识别真正主轴。

本轮只能围绕 1 个主因设计，不得把多个主因堆在一起。

# 第二步：题型选择与去重复约束
从下列主方式中只选 1 种；如必须组合，只允许“1 个主方式 + 1 个轻量辅助方式”：
1. 候选缺口二选一：给 2 个都像有用的补充事实，要求判断谁才是真正最小关键事实，并说明另一个为什么不够。
2. 子判断定位：先指出当前结论里到底是哪一个子判断还不能成立，再说明最少还缺什么。
3. 单步跳跃识别：要求指出“从哪一步跳到了哪一步”，而不是泛泛说证据不足。
4. 近似项分层：给 2-3 个都“不太能直接写结论”的说法，要求继续分高低，而不是只做可写/不可写二分。
5. 题干外补设识别：要求指出哪一个关键判断偷偷引入了题干外前提。
6. 单变量反事实：只改 1 个条件，要求说明哪一层判断因此变化，哪一层不变。
7. 具体化约束：要求答案必须绑定题干事实，不能退回套话。

去重复要求：
1. 如果同一样本上一轮已经是“最小关键事实/最小前提/最小跳步”题，本轮不得只换壳复用同一问法。
2. 如果上一轮已满分，本轮优先使用：
   - 候选缺口二选一
   - 子判断定位
   - 近似项分层
   而不是继续问“还缺哪一条”。
3. 如果上一轮已经是边界分类题，本轮不得再做同样的三分类换表述，除非原题唯一核心能力本来就是证据层级判断。

# 第三步：复杂度预算
1. 新题必须只有 1 个清晰主轴。读者应能用一句话说清“本题到底考什么”。
2. 新增事实或场景条件最多 3 条。
3. 输出任务最多 2 个；优先 1 个。
4. 候选项最多 3 个；如果是二选一题，优先只给 2 个近似候选。
5. 不得要求大表格、多层标签体系、固定句数、复杂编号系统。
6. 新题长度建议控制在原题的 3-8 倍；若原题很短，原则上不超过 900 个中文字符。
7. 如果上一轮已经很长，本轮优先删约束、收主轴，不要继续加材料。

# 第四步：可回答性与“唯一正确点”自检
生成新题前必须确保：
1. 题干已提供完成任务所需事实，不得要求靠题外专业知识作实体判断；
2. 如果一个问题存在 2 个以上都合理的“最小缺口”，不要做填空式问答，改成候选项区分题；
3. 不得把 rubric 的暗规则藏在题外，题面必须让人知道到底在比较哪一层；
4. 不得设置只能靠猜、靠经验、靠法律评价或靠主观常识才能回答的陷阱；
5. 新题要让弱模型主要在一个主错误上失分，而不是因为任务太多或格式太复杂而广泛失分；
6. 如果删掉题目中的一段要求后，核心考点明显变了，说明本题混入了多个能力点，必须继续简化。

# 禁止的进化方式
1. 禁止把难度主要建立在长篇格式、表格、固定模板或复杂编号上。
2. 禁止把“最小关键事实 / 最小前提 / 最小跳步”三种问法反复换壳当作题型创新。
3. 禁止一次性加入多个阶段、多套反事实、多组排序。
4. 禁止让题目主要考“遵循复杂指令”，而不是考原题对应的专业判断或推理能力。
5. 禁止在结论可拆成多个子判断时，直接问一个含糊的“最少还缺什么”，导致答案与 rubric 只能靠暗约定对齐。
```
