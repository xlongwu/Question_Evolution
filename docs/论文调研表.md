# 可调研的论文方向
一、最直接相关，优先看这组最接近你现在的项目。
  1. WizardLM: Empowering Large Language Models to Follow Complex Instructions
     (https://arxiv.org/abs/2304.12244)
     看点：Evol-Instruct 的原始代表作。重点看它怎么定义“进化”、怎么控制复杂度、怎么逐步把简单指令改成复杂指令。
  2. Automatic Instruction Evolving for Large Language Models
     (https://arxiv.org/abs/2406.00770)
     看点：Auto Evol-Instruct。和你项目很像，因为它已经在研究“进化策略本身如何自动选择和迭代优化”。
  3. Self-Instruct: Aligning Language Models with Self-Generated Instructions
     (https://arxiv.org/abs/2212.10560)
     看点：自举生成、过滤、去重、回灌训练。虽然不是“难化题目”本身，但它是你这个项目做“自动合成数据”的底层祖师爷。
  4. AutoBencher: Creating Salient, Novel, Difficult Datasets for Language Models
     (https://arxiv.org/abs/2407.08351)
     看点：不是进化单题，而是“自动搜索更难、更有新意、更有区分度的数据集”。这和你项目“不是机械改难，而是要挖出真实能力边界”高度相关。

二、对你最有启发的机制论文
这组解决的是“链式还是树状”“经验怎么复用”“怎么从失败中迭代”。
  5. Tree of Thoughts: Deliberate Problem Solving with Large Language Models
     (https://arxiv.org/abs/2305.10601)
     看点：为什么不能只走单一路径。你项目里“链式主流程 + 局部树状探索”基本就该从这里借鉴。
  6. Promptbreeder: Self-Referential Self-Improvement Via Prompt Evolution
     (https://arxiv.org/abs/2309.16797)
     看点：不是进化题，而是进化“进化prompt”。很适合你后面研究 evolution prompt 本身如何自动优化。
  7. Reflexion: Language Agents with Verbal Reinforcement Learning
     (https://arxiv.org/abs/2303.11366)
     看点：如何把失败经验写成可复用的语言记忆。对你项目里的“经验库 / operator memory bank”特别有用。
  8. STaR: Bootstrapping Reasoning With Reas
     oning (https://arxiv.org/abs/2203.14465)
     看点：如何利用模型自己生成的推理再训练自己。重点不是数学任务本身，而是“正确轨迹如何沉淀”。
  9. V-STaR: Training Verifiers for Self-Taught Reasoners
     (https://arxiv.org/abs/2402.06457)
     看点：如何利用“正确和错误样本”训练verifier。对你项目里的 judge / rubric / 评分稳定性问题很相关。

三、和“能力边界样本筛选”最相关这组最值得你用来回答“哪些样本值得进化”。
  10. HS-STAR: Hierarchical Sampling for Self-Taught Reasoners via Difficulty Estimation and Budget Reallocation
     (https://arxiv.org/abs/2505.19866)
     看点：非常贴你项目。核心观点就是：最有学习价值的是靠近模型能力边界的问题，不是太简单也不是太难。
  11. Self-Consistency Improves Chain of Thought Reasoning in Language Models
     (https://arxiv.org/abs/2203.11171)
     看点：同一题多条推理路径的价值。你后面如果要判断“这题是模型真不会，还是单次生成不稳定”，这篇很有用。
  12. Automatic Model Selection with Large Language Models for Reasoning
     (https://arxiv.org/abs/2305.14333)
     看点：不是所有题都该走同一种解法。对你后面做“样本类型 x 进化算子路由”有参考价值。

四、和你“参数量 vs 可训练边界”疑问最相关这组用来回答你刚才那个核心问题：边界到底是训练问题还是规模问题。

  13. Large Language Models are Zero-Shot Reasoners
     (https://arxiv.org/abs/2205.11916)
     看点：有些边界并不是参数绝对不够，而是提示方式没把能力调出来。
     oners: A Mechanistic Journey to the Edge of Generalization
     (https://arxiv.org/abs/2405.15071)
  15. Large Language Models have Intrinsic Self-Correction Ability
     (https://arxiv.org/abs/2406.15673)
     看点：如果换提示或自修正就能改善，那这类边界不一定是参数硬上限。

  建议阅读顺序
  如果只想高效调研，按这个顺序：

  1. WizardLM
  2. Auto Evol-Instruct
  3. Self-Instruct
  4. AutoBencher
  5. Tree of Thoughts
  6. Reflexion
  7. STaR
  8. HS-STAR
  9. V-STaR
  10. Grokked Transformers are Implicit Reasoners
