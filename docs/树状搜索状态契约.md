# 树状搜索状态契约

本文档是“树状搜索与回溯式多边界探索”阶段01的落地契约。当前阶段只定义字段、默认值、兼容策略和最小样例，不实现画像、路由、候选生成、边界去重或 frontier 调度。

## 1. 兼容原则

现有主流程仍以单链 JSONL record 为主。树状搜索字段先作为 `evolution_state` 的扩展字段存在，不替代 `prompt`、`meta_info`、`rubric`、`score_prompt`、`scoring_result`、`sample_profile`、`operator_route`、`candidate_selection`、`effect_analysis` 等现有字段。

缺少树状搜索字段的旧 record 可通过 `search_state_contract.normalize_search_state()` 补齐默认值。补齐结果不写回原文件，除非后续阶段明确调用。

默认回退路径是 `ENABLE_TREE_SEARCH=false`，因此现阶段不会改变 `run_loop.sh` 的单链执行语义。

## 2. evolution_state 扩展字段

| 字段 | 类型 | 默认值 | 含义 |
| --- | --- | --- | --- |
| `search_root_id` | string | `sample_<sample_id_or_index>_root` | 单样本搜索树根节点 ID |
| `current_node_id` | string | 根节点或当前 prompt hash 节点 | 当前题目节点 ID |
| `parent_node_id` | string/null | 根节点为 null，旧 evolved 记录默认指向 root | 当前节点的父节点 |
| `branch_id` | string | `main` | 分支 ID |
| `boundary_axis` | string/null | metadata 中的 axis 或 null | 当前分支目标能力轴 |
| `branch_status` | string | `exploring` | `exploring`、`boundary_hit`、`exhausted`、`duplicate`、`invalid` |
| `search_depth` | int | 原始题 0，旧 evolved 题 1 | 当前节点深度 |
| `max_search_depth` | int | 3 | 单分支最大深度 |
| `branch_budget_remaining` | int | `max_search_depth - search_depth` | 当前分支剩余扩展预算 |
| `sample_budget_remaining` | int | 10 | 单样本剩余候选预算 |
| `discovered_boundaries` | array | `[]` | 已发现边界点集合 |
| `recommended_next_axes` | array | `[]` | 后续推荐能力轴 |
| `already_explored_axes` | array | `[]` | 已尝试能力轴 |
| `no_new_boundary_rounds` | int | 未强制写入 | 连续无新增边界轮数 |
| `stop_status` | string | `continue` | 兼容旧停止状态，并新增树状搜索停止状态 |

新增 `stop_status` 包括：

- `continue_branch_search`
- `stop_branch`
- `branch_exhausted`
- `stop_sample`
- `sample_budget_exhausted`
- `sample_boundary_limit_reached`
- `no_new_boundary_stop`

旧状态继续保留，包括 `continue`、`effective_boundary_sample`、`stable_high_score_stop`、`local_tree_search_needed` 等。

## 3. search_graph.jsonl 行格式

每行表示一个搜索节点，schema 文件为 `schemas/search_graph_node.schema.json`。

必需字段：

- `sample_id`
- `search_root_id`
- `node_id`
- `branch_id`
- `depth`
- `prompt`
- `is_boundary_hit`
- `selected_into_mainline`
- `dedup_signature`

建议字段：

- `parent_node_id`
- `operator_used`
- `boundary_axis`
- `score_rate_before`
- `score_rate_after`
- `effect_label`
- `selected_as_boundary_leaf`
- `discard_as_duplicate`

定位：`search_graph.jsonl` 是搜索轨迹文件，不替代主链输出。主链 record 仍由现有阶段产物承载。

## 4. active_frontier.jsonl 行格式

每行表示下一步待扩展的 source node，schema 文件为 `schemas/active_frontier.schema.json`。

必需字段：

- `sample_id`
- `search_root_id`
- `frontier_node_id`
- `source_node_id`
- `source_node_type`
- `source_prompt`
- `prompt_source`
- `action_type`
- `branch_id`
- `target_boundary_axis`
- `search_depth`
- `next_depth`
- `max_search_depth`
- `branch_budget_remaining`
- `sample_budget_remaining`

`source_node_type` 取值：

- `current`
- `root`
- `parent`

`action_type` 取值：

- `expand_current_branch`
- `fork_from_root`
- `fork_from_parent`
- `stop_branch`
- `stop_sample`

阶段01只定义格式与样例，真正生成、消费和更新 frontier 留到阶段05。

## 5. 默认预算参数

默认值集中定义在 `search_state_contract.TREE_SEARCH_CONFIG_DEFAULTS`：

| 参数 | 默认值 | 含义 |
| --- | --- | --- |
| `ENABLE_TREE_SEARCH` | `false` | 保持旧单链流程为默认回退路径 |
| `MAX_SAMPLE_BRANCHES` | 4 | 单样本最多分支数 |
| `MAX_SAMPLE_DEPTH` | 3 | 单分支最大深度 |
| `MAX_SAMPLE_BOUNDARIES` | 3 | 单样本最多边界点 |
| `MAX_SAMPLE_CANDIDATES_TOTAL` | 10 | 单样本总候选预算 |
| `MAX_NO_NEW_BOUNDARY_ROUNDS` | 2 | 连续无新增边界停止阈值 |
| `MAX_GLOBAL_NEW_BOUNDARY_GAP` | 2 | 全局无新增边界停止阈值 |
| `ENABLE_BRANCH_BACKTRACK` | `true` | 允许从父节点回溯开分支 |
| `ENABLE_ROOT_FORK` | `true` | 允许从根节点开新分支 |

## 6. 最小样例

阶段01提供两个最小 JSONL 样例：

- `tests/fixtures/stage01_search_graph.jsonl`
- `tests/fixtures/stage01_active_frontier.jsonl`

样例覆盖根节点、子分支、边界命中、重复分支、当前节点扩展、根节点 fork 和父节点 fork。
